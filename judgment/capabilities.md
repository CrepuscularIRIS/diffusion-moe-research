# Capability Bank — the positive record of what the loop CAN do (the evolving layer)

> The positive twin of the Failure Atlas: lessons record calibration corrections, capabilities record
> what the loop has PROVEN it can do. Operations + curation = the research-os `ledger` skill. Write a
> row at each `CLAIM_STANDS` or surprising verified PASS. **`/prospect` Mine 2 reads this file** — an
> established capability is a problem seed (what does it now make possible?). Re-grade solidity as
> evidence accrues (DISCOVERED → CONFIRMED → MECHANISTIC); a capability contradicted by a later failure
> is demoted or bounded, never silently kept. Curate at every programme pulse.

## Row schema

```
| capability | minimal conditions | corrosion check (DISCOVERED/CONFIRMED/MECHANISTIC) | boundary | transfer targets |
```
- **capability** = what the loop established it can do (a verified result / a working recipe / a
  reproduced env).
- **minimal conditions** = the smallest setup under which it holds.
- **corrosion check** = solidity: DISCOVERED (once) → CONFIRMED (reproduced) → MECHANISTIC (the WHY is
  understood, so transfer is predictable).
- **boundary** = where it stops.
- **transfer targets** = adjacent problems it might unlock.
