Consecutive denoising steps may violate that through smooth state evolution.\n\nMitigations:\n\nuse coarse pre-registered buckets, not hundreds of exact timesteps, for the primary test;\n\nvalidate with autocorrelated nulls;\n\nreport block/circular-shift permutation sensitivity;\n\ncluster-bootstrap CIs at trajectory or prompt level;\n\navoid claiming causal timestep use unless you run timestep-swap interventions.\n\n4. Load balancing and capacity constraints\n\nTraining-time load balancing can flatten global usage. Inference-time capacity constraints can introduce artificial dependence through batch composition or timestep-specific token load.\n\nPrimary analysis should use pre-capacity router top-k choices. If only post-capacity dispatch is available, log capacity overflow and batch composition, and include a sensitivity analysis with batches fixed or with capacity-free reruns.\n\n5. Token state evolves with timestep\n\nPassive logs cannot fully separate “router specializes to timestep” from “router specializes to token/canvas states that evolve with timestep.”\n\nFor the main paper, phrase the passive-log result as:\n\nexpert usage is conditionally time-aligned within fixed prompt/canvas/position tracks.\n\nFor a stronger mechanistic claim, add an intervention:\n\nfreeze the same canvas state,\n\nreplay the router with different timestep embeddings,\n\ntest whether expert selection changes with the injected timestep.\n\nThat intervention directly targets timestep conditioning.\n\n6. Bucket choice can become hidden multiple testing\n\nIf you try K=4,8,16, early/mid/late splits, log-SNR splits, and exact-t splits, that is multiple testing. Either:\n\nchoose one bucketization on a pilot set,\n\npre-register one primary bucketization,\n\nor include all bucketizations in the global max-statistic correction.\n\n7. Sparse expert counts\n\nWith 128 experts and many buckets, some expert-bucket cells will be sparse. That is acceptable because the p-value is permutation-calibrated. Do not rely on asymptotic χ\n2\n p-values.
