"My pick: Predictive Self-Conditioning Bridge — transport the model’s belief state, not its tokens\n\nI would not spend the next experiment on another token-level sampler, verifier router, output observer, or residual re-denoiser. The one upstream lever that still looks live is:\n\nLearn or oracle-test a tiny on-manifold bridge that advances the native self-conditioning distribution forward in denoising time, then feeds that advanced belief through DiffusionGemma’s already-trained self-conditioning gate.\n\nThis is primarily a few-step speedup bet. I am skeptical that a frozen near-ceiling model has much accuracy headroom unless the same probe shows that verified-positive self-conditioning can causally rescue wrong trajectories.\n\nThe key reason this is the right interface is that DiffusionGemma already has a trained cross-step bottleneck: each denoising step takes the current canvas plus a self-conditioning matrix, passes the self-conditioning through a gated MLP, and adds it to token embeddings before the transformer; its self-conditioning matrix is computed from the previous step’s token distribution embedded back through the tied token embedding space. \narXiv\n DiffusionGemma also runs fixed 256-token canvas blocks with up to 48 denoising steps under entropy-bound sampling, so cutting denoising NFEs without touching the backbone is the only plausible speed frontier. \nvLLM Recipes\n\nThis is the self-conditioning analogue of “predict the future state variable,” but the state variable is not Euclidean token embedding flow. It stays on the probability simplex and enters only through the native softmax(logits) · W · scale channel.\n\n1. Exact operator / adapter\n\nCall it PSC-Bridge: Predictive Self-Conditioning Bridge.\n\nNative update at step k, position i:\n\np\nk,i\n\t​\n\n=softmax(ℓ\nk,i\n\t​\n\n/T\nk\n\t​\n\n),m\nk,i\n\t​\n\n=scale⋅p\nk,i\n\t​\n\nW.\n\nPSC-Bridge replaces only the stored self-conditioning belief for the next forward pass:\n\np\n~\n\t​\n\nk,i\n\t​\n\n∝p\nk,i\n1+α\nk,i\n\t​\n\n\t​\n\np\nk−1,i\n−α\nk,i\n\t​\n\n\t​\n\n\nequivalently,\n\nlog\np\n~\n\t​\n\nk,i\n\t​\n\n=logp\nk,i\n\t​\n\n+α\nk,i\n\t​\n\n(logp\nk,i\n\t​\n\n−logp\nk−1,i\n\t​\n\n)−logZ.\n\nThen feed:\n\nm\n~\nk,i\n\t​\n\n=scale⋅\np\n~\n\t​\n\nk,i\n\t​\n\nW\n\ninto the existing self-conditioning gated MLP at the next denoising step.\n\nCrucially:\n\nDo not use \np\n~\n\t​\n\n for the current token accept/sample decision.\n\nDo not change EntropyBoundSampler.accept_canvas.\n\nDo not move canvas embeddings continuously.\n\nDo not snap Euclidean vectors to nearest tokens.\n\nDo not train or touch the backbone, router, MoE experts, tied embedding matrix, or LM head.\n\nThe only learned object is a scalar α\nk,i\n\t​\n\n per position:\n\nα\nk,i\n\t​\n\n=α\nmax\n\t​\n\n⋅σ(g\nϕ\n\t​\n\n(f\nk,i\n\t​\n\n)),\n\nwhere g\nϕ\n\t​\n\n is a tiny MLP, for example 2 layers, width 32 or 64, under 20k parameters. This keeps the intervention in the distributional regime the native self-conditioning channel has plausibly seen.\n\nWhy this operator: the e-geodesic extrapolation\n\np\n~\n\t​\n\n∝p\nk\n1+α\n\t​\n\n/p\nk−1\nα\n\t​\n\n\nis the probability-simplex version of a secant/momentum step. It asks: “where is the model’s own clean-token posterior already moving?” Then it feeds that advanced posterior mean embedding into the trained SC gate. That is the right geometry for a tied-embedding discrete dLLM; the state is a belief distribution, not an embedding-space particle.\n\nTraining target, only after the oracle passes:\n\nL\nPSC\n\t​\n\n=\n\t​\n\np\n~\n\t​\n\nk,i\n\t​\n\nW−sg(p\nk+Δ,i\nteacher\n\t​\n\nW)\n\t​\n\n2\n2\n\t​\n\n+λD\nKL\ntopK\n\t​\n\n(p\nk+Δ,i\nteacher\n\t​\n\n∥\np\n~\n\t​\n\nk,i\n\t​\n\n).\n\nUse native 48-step trajectories as teacher traces. For speed, no verifier labels are needed. For a math-accuracy variant, weight only trajectories whose final answer is verified correct, but I would not start there.\n\n2. If this fails, the scalar bridge is dead even if the direct SC oracle works.\n\nMetric\n\nPrimary metric:\n\nVerifiedAcc\n24\n\t​\n\n\nunder math_verify, compared pairwise against native truncated-24 and native full-48.\n\nSecondary metric:\n\nrescue\n24\n\t​\n\n=Pr[PSC-24 correct∣native-24 wrong, native-48 correct].\n\nAlso report mean NFEs. Novelty slice vs the overlap territory\n\nThis is not D3PM novelty; D3PM already established structured discrete corruption processes, including uniform and absorbing-state variants. \narXiv\n It is also not a new discrete guidance derivation; classifier/classifier-free guidance for discrete diffusion has already been derived, especially for uniform-noise models. \narXiv\n\nThe defensible slice is narrower and cleaner:\n\nFrozen-backbone self-conditioning time transport: a tiny probability-simplex bridge that advances the model’s native self-conditioning belief state, while leaving the token sampler and denoiser unchanged.\n\nAgainst the named overlaps:\n\npMF / partial masking / Prime. Partial masking changes the discrete state space and training objective by introducing intermediate masked/unmasked states. PSC-Bridge does not alter the forward kernel, state alphabet, corruption process, or backbone training. \narXiv\n\nMeanFlow / reflow. MeanFlow learns average velocity for one-step continuous generative modeling. PSC-Bridge does not learn a continuous data-space velocity and does not train a new generator; it only predicts a future self-conditioning posterior mean inside an already-trained discrete dLLM. \narXiv\n\nJiT / ELF / DSL-LLaDA. ELF moves language generation into continuous embedding-space flow and maps back to discrete tokens at the end; DSL-LLaDA adapts a masked DLM to continuous per-token noise and SDE-style inference. PSC-Bridge keeps the canvas discrete and touches only the native SC distribution pW, not an embedding-flow canvas. \narXiv\n+1\n\nSDTT / DiDi. SDTT and DiDi are distillation methods for faster dLLM generation; DiDi explicitly distills a few-step student initialized from a pretrained dLLM and reports up to 64× acceleration. PSC-Bridge is not a student model and does not distill the denoiser; the frozen generator remains the generator, and the only learned object is a tiny bridge for its existing SC state. \narXiv\n+1\n\nLRD. LRD maintains distributional mixtures and uses a predictive feedback loop to improve parallel decoding, reporting reasoning/code gains and speedups. PSC-Bridge is not a replacement decoding interface: it does not use mixtures as canvas tokens, does not change finalization policy, and does not touch current-token selection. It acts one layer earlier, at the trained SC injection point. \narXiv\n\nLoopholing. Loopholing adds a deterministic latent pathway to preserve distributional information across categorical sampling. PSC-Bridge exploits the fact that DiffusionGemma already has such a trained self-conditioning pathway; the novelty is not “add a bypass,” but “time-advance the existing bypass under a frozen production model.” \narXiv\n\nBiggest overclaim risk: oracle future SC can leak trajectory-specific solution information. A positive direct-oracle result only proves controllability of the SC channel, not learnability. That is why the scalar-grid oracle is important. If the direct oracle works but the scalar oracle does not, the result is scientifically interesting but not enough to justify claims of a practical tiny adapter.\n\n4. Honest stance on accuracy headroom\n\nI would not bet first on a frozen-adapter accuracy gain on AIME/MATH-L5. Your negatives strongly suggest that post-hoc computation over the generator’s own outputs is already on the native Pareto surface. AIME/MATH-L5 errors are likely not mostly “late token uncertainty” errors; they are probably wrong latent proof/search trajectories that the frozen denoiser has already committed to representationally.\n\nThe minimal result that would change my mind is a separate verified-positive SC donor oracle:\n\nsample multiple full native trajectories per hard problem;\n\nfind a verified-correct donor trajectory and a verified-wrong recipient trajectory for the same problem;\n\nreplay the wrong trajectory from step 12, 18, or 24 while injecting only the correct donor’s future SC embeddings, not its tokens;\n\nrequire wrong→correct rescue ≥15% on AIME/MATH-L5, while wrong-donor, other-prompt donor, and position-shuffled donor stay below 3%.\n\nIf that fails, I would state the negative result plainly: for this frozen near-ceiling dLLM, neither post-hoc routing nor native-SC upstream steering exposes meaningful verified-accuracy headroom; only actual denoiser retraining / larger task finetuning / architecture-level transport changes move the frontier.\n\nBut I would still run PSC-Bridge first, because it is the cleanest remaining falsifiable lever: one trained interface, one probability-geometric operator, one cheap oracle, and a real possible outcome—48-step quality at ~24 NFEs."
