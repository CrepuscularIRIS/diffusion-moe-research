## 1. The three things that MUST survive (the user's own priority order)
1. **★ THE ONE INVARIANT** — *"A verdict that helps the proposer if gamed must never be granted by the
   proposer."* The proposer self-administers **DOWN verdicts only** (kill / downgrade); `CLAIM_STANDS` needs
   an **independent substrate** (another model's fresh context / the Codex hook / Pro); a **contribution
   terminates in the human**. Every command orbits this. It is the safety core — non-negotiable.
2. **★ /forge — the value engine.** One problem → 3–5 candidate approaches, each forced through the axes:
   a **rival-school** candidate (methodology diversity) · a **non-incumbent-frame** candidate (anti-math-
   monoculture) · a **taste-operator** candidate (object-shift move) · a **DIFF-PREDICTION** field
   (*no differential prediction ⇒ relabeling ⇒ KILL*). Plus the **REGENERATION RULE** (the backlog regrows
   from each failure — never an up-front menu). This forcing is ESSENCE, not ceremony — it is what breaks
   the model's monoculture. Keep it.
3. **★ THE TASTE MODEL** (`skills/forge/references/taste.md`): taste = prior over info-per-effort;
   **generative taste ("what would make this good?") adds value, evaluative taste ("is this good?") only
   subtracts**; the **unit of progress is a SURPRISE — an observation that changed the plan** (unfarmable by
   orderly retreat, unlike v0.4 progress tokens).
