有，而且这些原则通常不是“故意不说”，而是因为顶级工作者已经内化了，觉得理所当然；但一般人并不会默认执行。

最核心的一句是：

**顶级工作者默认不相信自己，普通工作者默认相信自己。**

这句话展开后，基本就是所有隐藏原则的来源。

### 1. 好结果默认是假的，直到被反复打穿

一般人看到结果涨了，第一反应是高兴；顶级工作者第一反应是怀疑。

他们会默认问：

“是不是 baseline 太弱？”

“是不是 seed 偶然？”

“是不是 data leakage？”

“是不是 metric 有 bug？”

“是不是 eval protocol 不公平？”

“是不是一个 trivial baseline 也能做到？”

这在文件里写得很明确：强经验研究者会主动试图摧毁自己的好结果，检查 train/test contamination、metric implementation、trivial baseline 是否吃掉大部分收益。

隐藏原则是：**在你自己无法反驳你的结果之前，不要急着让别人相信。**

很多人不遵守，是因为他们潜意识里想“保住好结果”。顶级工作者想的是“如果它是假的，越早发现越便宜”。

### 2. Baseline 不是陪衬，而是敌人

普通工作者常见做法是：自己的方法精调很多轮，baseline 用论文默认参数或别人 repo 里的配置。

顶级工作者的默认原则是：**baseline 必须被调到足够强，强到你希望它输、但你又必须承认它已经被公平对待。**

文件里说得很硬：如果你的提升在 baseline 获得同等调参预算后消失，那你就没有东西。

这是一条很多人不会遵守的隐性纪律。因为强 baseline 会“威胁”自己的 idea。但顶级工作者知道，弱 baseline 只会制造虚假信心，最后在审稿、复现、后续工作里被摧毁。

### 3. 不许用复杂度掩盖无效性

普通工作者喜欢复杂方法，因为复杂看起来像创新。

顶级工作者默认相反：**复杂度是债务，简单性是武器。**

文件里提到，简单不是审美，而是 selection criterion：简单方法更容易 ablate、更容易复现、更难被质疑为 tuning artifact，也更容易成为别人继续构建的平台。

隐藏原则是：如果一个想法不能用两三句话讲清楚，很可能不是深刻，而是混乱。

当然，有些问题确实需要复杂系统。但顶级工作者会先问：“这个复杂性是必要的吗？有没有更小的机制能解释主要收益？”

普通人经常把复杂当护城河。顶级工作者把复杂当嫌疑犯。

### 4. 项目一开始就写 kill criteria

很多人做项目是这样的：先试试，做着做着投入越来越多，最后因为 sunk cost 不愿意停。

顶级工作者会反过来：**在还没爱上这个项目之前，先写下什么证据会杀死它。**

文件里说，想法应该带有 cheap kill criteria；很多 idea 死在文档里，这是正常的。项目也应有 time-boxed gate：两周看 first signal，6–8 周看是否值得继续，不是问“我喜不喜欢”，而是问“新证据是否增加了”。

隐藏原则是：**努力不算资产，证据才算资产。**

这是很难做到的，因为人会把投入过的时间当作理由。顶级工作者更冷酷：没有 evidence accumulation，就 kill 或 park。

### 5. 每个实验必须提前写预测

普通人跑实验，经常是：“先跑一下看看。”

顶级工作者默认不这么做。他们会在实验前写：

“I expect X because Y.”

如果结果符合预期，说明当前模型暂时成立；如果结果不符合预期，要么系统有 bug，要么你的理论错了，要么出现了更有价值的 surprise。

文件里说，强实验室和弱实验室的区别之一，就是是否在实验前写下假设和预测；否则就会变成“run stuff, look at numbers, tell a story afterward”。

隐藏原则是：**没有预测的实验，信息含量很低。**

AI 时代这更重要。因为 AI 可以帮你快速生成很多实验，但如果没有 hypothesis，它只会帮你更快地制造噪声。

### 6. 不要先造大系统；先证明小系统能学到东西

普通人容易一上来搭完整 pipeline、完整 agent、完整模型、完整 benchmark。

顶级工作者默认先做最小可行实验：tiny batch overfit、initial loss sanity check、data visualization、toy case、small-scale ablation。

文件里提到 Karpathy 式工作流的核心是 paranoid, incremental verification：先确认 pipeline 能学到任何东西，不要写完整系统后再端到端 debug。

隐藏原则是：**scale amplifies both signal and bugs。**

大规模不是用来寻找真相的第一步，而是用来验证已经在小规模下干净成立的信号。

### 7. 写作不是包装，是研究工具

普通人把写论文当成最后阶段的事情。

顶级工作者默认很早写 paper skeleton：abstract、claim、main table、ablation table、Figure 1。文件里明确说，写 paper skeleton 会迫使你回答：我要证明什么？哪张表能证明？Reviewer 第一个质疑是什么？每个实验要么填一个关键 cell，要么被砍掉。

隐藏原则是：**写不清楚，通常不是语言问题，而是思考问题。**

很多人以为自己“实验做完了，只差写”。顶级工作者知道，如果写不成一个干净故事，实验本身可能就没有形成清晰 claim。

### 8. 一篇工作只允许一个核心 claim

普通人喜欢把很多贡献塞进一篇论文里，因为怕“不够”。

顶级工作者默认压缩：one claim per paper。

文件里说，paper narrative 应该是 claim-first，不是 chronology-first；论文不是你做了什么的流水账，而是从读者关心的问题到你能证明的 claim 的最短路径。多 claim 会稀释审稿分数和影响力。

隐藏原则是：**论文不是展示劳动量，而是建立一个新事实。**

这点对研究、产品、工程报告都通用。顶级输出通常不是“东西很多”，而是“主张极清楚”。

### 9. 会议不是汇报进度，而是审判证据

普通团队开会常见内容是：“我这周做了 A、B、C，下周准备做 D。”

强团队默认不是这样。文件里提到，高质量团队的同步通常围绕 plot、table、evidence，而不是 status narrative；当前决策和假设写在 shared doc 里。

隐藏原则是：**没有图、表、错误样本、实验记录，就不要用故事占据会议。**

一般人用会议证明自己在努力。顶级工作者用会议压缩不确定性。

### 10. 内部 review 要比外部 review 更残酷

普通人怕别人提前批评，想等“更完整”再给别人看。

顶级工作者默认提前暴露。他们会让同事扮演 Reviewer 2，攻击 baseline、claim-evidence gap、missing ablation。文件里也提到，强实验室会在投稿前 2–4 周做 adversarial internal review。

隐藏原则是：**真正的批评越早来，越便宜；越晚来，越致命。**

很多人不遵守，是因为他们把批评当自尊打击。顶级工作者把批评当成本最低的梯度信号。

### 11. 阅读不是为了“知道”，而是为了“能打”

普通人读论文容易变成收集信息，读了很多但不改变自己的判断。

顶级工作者默认 shallow-wide, deep-narrow：大范围扫领域趋势，只对自己要 build on 或 beat 的论文深读，甚至复现关键部分。文件里也说，真正高产的人不会完整读大多数论文，而是对关键论文进行 adversarial reading 和 reimplementation。

隐藏原则是：**读论文的目标不是理解作者，而是更新你的研究地图。**

读完一篇关键论文，应该能回答：

“它真正解决了什么？”

“它没解决什么？”

“它的结果靠什么假设成立？”

“我如果要打败它，最脆弱的位置在哪里？”

### 12. 想法主要来自摩擦，不是头脑风暴

普通人会问：“怎么产生好 idea？”

顶级工作者更重视“把自己放在会产生摩擦的位置”：复现、debug、看错误样本、教别人、重写代码、跑失败实验、和强同行争论。

文件里有一句很关键：productive researchers 的好想法经常来自实验中某些东西坏了或表现异常，来自 teaching/reimplementing/conversations，而不是纯 literature gap analysis。

隐藏原则是：**不要坐着等灵感，要让系统不断和现实碰撞。**

AI 可以帮你 brainstorm，但真正有价值的 idea 往往来自“为什么这个东西不按理论运行”。

### 13. 决策必须写下来，不要相信记忆

普通人习惯口头讨论、脑中记计划、临时解释实验。

顶级工作者默认 written decision log：当前假设、实验配置、为什么跑、预期是什么、结果是什么、下一步为什么这样选。

文件里提到，强团队会让 decisions and current hypotheses live in a running doc，shared experiment logs 和 dashboards 让所有人直接检查证据。

隐藏原则是：**没有记录，就没有科研记忆；没有科研记忆，就无法复利。**

很多人低效不是因为不努力，而是同一个判断错误重复犯，自己还意识不到。

### 14. 真正的目标不是赢当前 leaderboard，而是通过未来 test set

普通人容易优化眼前指标：当前 SOTA、当前 benchmark、当前 review preference。

顶级工作者会问：“这个问题两三年后还重要吗？如果规模、数据、模型、使用场景变化，这个 claim 是否仍成立？”

你给的 GPT.md 里提到一个很关键的表述：真正的 test set 是未来，研究要避免对当前 leaderboard 和当前设定过拟合。

隐藏原则是：**当前 benchmark 是训练集，不是世界本身。**

这解释了为什么顶级工作者常常不是只追 1% 提升，而是选择能改变问题 framing、evaluation、assumption 的工作。

### 15. 他们默认“产出质量”来自环境，不只来自个人意志

普通人喜欢把顶级产出归因于天赋或自律。

顶级工作者更知道：peer quality、infrastructure、compute、internal review、reading group、shared codebase、强 baseline 文化，都会改变一个人的上限。

文件最后也提醒，FAIR、DeepMind、OpenAI 等环境的强产出部分来自 compute、infrastructure 和 dense peer quality；个人只能部分替代这些输入，但可以通过 reading group、paper swap、人为制造 peer pressure 来近似。

隐藏原则是：**不要只优化自己，要优化自己所在的反馈环境。**

一个人在弱反馈环境里，很容易高估自己。顶级环境最残酷也最有价值的地方，是它不断校准你的标准。

---

我会把这些隐藏原则压缩成一句操作性极强的话：

**顶级工作者默认把自己当成最危险的误差源，所以他们设计流程来约束自己：约束选题、约束实验、约束解释、约束写作、约束投入、约束自尊。**

很多一般工作者的问题不是不会努力，而是太相信自己的努力、直觉和好结果。顶级工作者也有直觉，但他们不会让直觉裸奔；他们会让直觉进入一个高压、可证伪、可复现、可被攻击的系统。
