结论：**这个 zip 里的 `research-os` 不是最终 v1.0，而是一个“方向正确但已经重新膨胀”的 v0.9。** 目标文档的核心不是再加一版功能，而是收敛：保留 **one invariant / `/forge` / taste model** 三个不可切断核心，保留生成器的四轴强制，但删除围绕它长出来的仪式层。

我的最终判断如下。

## 应该保留的东西

**保留为一等核心：**

1. `/prospect`：问题生成器。
   但要变短，只保留五个 mine、own-log anomalies first、问题卡 `{Q, TYPE, WHY-NOW, STAKES, PROBE, SURPRISE}`。

2. `/forge`：价值引擎，不能砍。
   当前看起来复杂，但它不是仪式，是反单一模型先验的核心：rival-school、non-incumbent-frame、taste-operator、DIFF-PREDICTION、regeneration rule 都应保留。目标文档也特别警告不要误砍 `/forge` forcing。

3. `/adversary`：唯一 claim-boundary gate。
   这里承载 one invariant：提案者只能自授 DOWN verdict；`CLAIM_STANDS` 必须独立 substrate；contribution 终止于 human。这个是安全核心，不应分散到多个命令。

4. `taste.md`、`schools.md`、`frames.md`、`taste-operators.md`、`research-types.md`：保留为 generator palette。
   这些不是流程负担，而是生成器原料。目标也明确说五个 reference docs 是 essence。

5. Arbor substrate：保留 idea-tree、`eval_run`、merge/worktree 相关结构。
   重点是防止“凭记忆写数字”和 side-channel drift；Arbor execution engine 本身不属于 methodology cut。

## 应该删除或折叠的东西

我建议 v1.0 的 command surface 从 **11 个降到 5 个左右**：

| 当前命令                           | v1.0 处理                                                                 |
| ------------------------------ | ----------------------------------------------------------------------- |
| `/prospect`                    | 保留，压缩                                                                   |
| `/forge`                       | 保留，不能砍 forcing                                                          |
| `/autopsy`                     | 保留或降为 `/forge` 的失败分支；只留一句学习 + conversion law                            |
| `/prereg`                      | 保留但极简：metric/split、accept/kill、seeds/one-var                            |
| `/exp-verify`                  | 删除为独立命令；并入 run step：artifact read + anti-no-op                          |
| `/adversary`                   | 保留为唯一 claim gate                                                        |
| `/rigor`                       | 删除为命令；把 `UNIT + paired test + allowed claim envelope` 三行并入 `/adversary` |
| `/compass`                     | 删除为定期命令；变成持续反射：surprise accounting + type drift                         |
| `/epistemic-calibration`       | 删除；其核心已被 one invariant 覆盖                                               |
| `/irreversible-decision-audit` | 删除；cheap-falsifier-first + human boundary 并入核心规则                        |
| `/artifact-acceptance`         | 删除为命令；外部交付前作为 4 行 handoff checklist                                     |

`skills/rigor/references/evidence-contract.md` 也不应作为 v1.0 的 reference doc 保留。它可以被压成 `/adversary` 里的几条证据底线：**statistical unit、paired comparison、seed variance/claim scope、opposing baseline wins if possible**。否则它会继续诱导系统回到“证据合约/仪式产物”模式。

## v1.0 应该长什么样

我认为最终形态应是：

```text
research-os v1.0

1. Taste first
   taste = prior over expected information per effort.
   Progress = surprise that changed the plan.

2. Generate
   /prospect -> find problems
   /forge -> generate 3–5 candidates through 4 axes:
     schools · types · frames · operators
   Every candidate needs DIFF-PREDICTION.
   No differential prediction = relabeling = kill.

3. Run
   Before a claim-bearing run:
     metric/split · accept/kill · seeds/one-var.
   After run:
     read numbers from artifacts;
     verify intervention fired;
     no artifact, no number.

4. Claim
   /adversary is the only gate.
   Proposer may self-grant DOWN only.
   CLAIM_STANDS needs independent substrate.
   Contribution/paper promotion = human.

5. Learn
   Every null becomes:
     constraint · new candidate · or region-close.
   Same root cause killed 2–3 mechanisms => leave the region.

6. Anti-accretion
   When something breaks:
     first ask whether the agent failed to follow the simple rule;
     then ask whether an existing piece can absorb it;
     then ask what can be deleted;
     adding a mechanism is last resort and requires deleting another.
```

## 最重要的结论

**不要在当前 v0.9 上继续补丁式修改。** 这个仓库已经到了“需要做减法”的点。正确的 v1.0 不是 `v0.10`，而是把当前 11-command 系统压回一个小表面：**生成器 + 单一 claim gate + 极简 run discipline + Arbor 事实账本 + anti-accretion damping**。

我的建议是：下一步直接重写 README、plugin metadata、commands/skills 目录，把 v1.0 做成“少数强规则 + 五个 palette 文档”的收敛版；不要保留 `/rigor`、Layer-2、scheduled `/compass` 作为命令。
