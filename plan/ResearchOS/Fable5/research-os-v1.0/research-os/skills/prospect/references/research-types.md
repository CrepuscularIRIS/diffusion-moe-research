# Research types — the value axis

> Schools = *how to think*; types = *what kind of output, how it is valued, how it is verified*.
> Type-blindness — applying one type's gates (novelty) to all types — was the original framework's central
> bug: it killed honest improvement work for having neighbors and herded output toward certificates.
> **Rule zero: name the TYPE first; verify by type.** Every prospect card, forge card, and prereg carries a
> type; `/adversary` applies that type's checks, not a universal battery.

| Type | Value = | Verify by | Characteristic failure |
|---|---|---|---|
| **Improvement 刷分** | a measured Δ vs an honestly-tuned baseline — idea novelty is IRRELEVANT; the contribution IS the Δ | baseline fairness (equal budget, paired stats) + Δ-reality (≥3 seeds, controls) | unfair baseline; occupancy-as-veto; Δ that dies off-benchmark |
| **Evaluation 评估** | a DECISION that changes because of the measurement | name the downstream decision that now goes differently | **certificate-production** — the loop's most seductive false productivity; passes every rigor check, matters to no one |
| **Survey 综述** | a map that changes where people search + a mined problem list | ≥1 problem someone would drop current work to attack; says where NOT to dig | summarizing instead of mining; coverage worship |
| **Novelty 新对象** | a reframe that explains old failures AND predicts new ones; the object becomes vocabulary | differential prediction on EXISTING logs, with a shuffled-S control (where the pattern doesn't apply — optimization-dynamics ideas — say so and state the alternative kill) | the cosmetic relabel: an elegant story that predicts nothing |
| **Systems 系统** | an end-to-end wall-clock/cost lever at equal quality | end-to-end on the real path, never a proxy; equal quality on the sealed eval | kernel-2×-faster = 3%-end-to-end wins; unoptimized-baseline stacks |
| **Negative 证伪** | a killed assumption the community actually builds on | assumption load-bearing (cite who builds on it); scope stated exactly; a STRUCTURAL claim needs independent confirmation | strawman-killing; scoped null inflated to "direction dead" |
| **Theory 理论** | many phenomena → one mechanism, with ≥1 NEW prediction | the new prediction, tested or preregistered | post-hoc unification that predicts nothing |
| **Tooling 工具** | an experiment class others can now run — and do, within a cycle | used in anger on a real question; unused infra = 0 | infra for its own sake (the disease this framework itself had) |
| **Reproduction 复现** | trust recalibration on a LOAD-BEARING result | pre-state what differs and what a discrepancy means; sealed protocol | reproducing the peripheral |

## Rules of use
1. Declare the type on every card; carry it through `/prereg`; `/adversary` verifies by type.
2. **Type-drift is the canonical loop failure**: goal says design/improvement, artifacts trend
   evaluation-shaped — because eval work is the most gateable. Detect structurally: type the last 3–5
   artifacts against the goal's declared type (`/autopsy` reflex 2).
3. Types compose across a campaign: 综述 → problem list → improvement runs → maybe a novelty reframe.
4. The He-bar is type-relative (`taste.md` §4) — grade against the right column.
