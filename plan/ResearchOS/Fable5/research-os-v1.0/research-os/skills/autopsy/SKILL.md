# Autopsy — a dead run must feed the generator

Banking negatives is bookkeeping; the value of a failure is what it *generates*. This is a GENERATOR
command wearing a coroner's coat.

## Steps

### 1. Boring-first triage (always first — most "negative results" are bugs)
Bug? Data problem? Config? Insufficient scale? Broken harness? Did the intervention actually fire
(`/exp-verify` anti-no-op evidence)? Does the baseline reproduce its known number? Stable across seeds?
If boring → fix and re-run; nothing gets banked.

### 2. Mechanism-level why
Locate the failure INSIDE the `/forge` card's mechanism-sentence — not the symptom, but which link of the
causal story broke: wrong hypothesis (the load-bearing variable is misnamed)? right mechanism, this
implementation can't express it? an unstated assumption the data/regime doesn't have? Write:
`the mechanism assumed <X>; the run shows <evidence>; therefore <which link broke>`.

### 3. Honest scope grade (DOWN-only — the invariant applies here too)
`failed-attempt` (implementation-level) and `scoped-negative` (this config/scale/dataset) are
self-grantable. **`structural-negative`** ("the MECHANISM cannot work" — a citable claim) is
**independent-only** via `/adversary` C; self-grade caps at `scoped-negative (structural-PENDING)`. Never
inflate a scoped null to "direction dead" without the mechanism-level why.

### 4. THE CONVERSION LAW (mandatory — the point of this command)
Every autopsy emits **at least one** of:
- **(a) a CONSTRAINT** — a new fact about the search space, stated so the next `/forge` can use it; it
  re-prices every backlog candidate that violates it.
- **(b) a CANDIDATE** — a new/reshaped forge entry, derived from the why (execute the `/forge`
  REGENERATION RULE; run the relevant 10-question-generator entries on the RESULT). A genuinely fresh
  angle routes its design to the external brain — the same default as `/forge` step 7.
- **(c) a REGION-CLOSE** — "this closes the region because <X>". **Don't circle**: ≥2–3 mechanisms killed
  by the SAME root cause ⇒ the REGION is dead; the next move is `/prospect` on genuinely lateral
  territory, not another neighbor of the dead idea.

**A negative that produces none of these is not banked — it is an incomplete autopsy.**

### 4′. The ledgers (the generator's supply chains)
- **[ANOMALY]** — any observation you did NOT predict, positive or negative, goes to the tree node
  insight as one line **with raw numbers read from the artifact** (artifact-fidelity, never memory):
  `[ANOMALY] <observation> · <raw numbers> · <the frame that would model it>`. This feeds `/prospect`
  Mine 1 — own anomalies are the only off-distribution data we have; an anomaly banked but never mined is
  the canonical leak.
- **[OPERATOR-CANDIDATE]** — if the anomaly reveals a *reusable modeling MOVE*, draft a card with the
  three load-bearing fields + the deletion test (`../forge/references/taste-operators.md`). It enters the
  bank ONLY through the corrosion gate + an **INDEPENDENT audit** (the invariant — no self-granted
  operators). On FAIL it stays a scoped constraint.

### 5. Programme pulse (when a *direction* closes — the continuous reflexes, formerly /compass)
1. **Surprise accounting** — which observation changed the plan in the last N cycles? Zero surprises =
   farming process → force a generator move (`/prospect` on fresh corpus, or promote the deferred
   rival-school candidate).
2. **Type-drift** — type the last 3–5 artifacts against the goal's declared TYPE. Eval-shaped drift under
   a design goal is an incentive gradient, not a discipline failure; name the next ON-type artifact and
   dispatch it.
3. **Lakatos health** — hard core intact (failures hitting auxiliaries, not the core)? positive heuristic
   still ADDING candidates (a monotonically shrinking backlog is a menu)? still predicting new facts, or
   only absorbing failures with patches?
4. **Process budget** — workflow/meta >20% of effort sustained ⇒ stop building process, ship a run.
5. **Diversity** — across the last N `/forge` rounds, ≥2 distinct frames and ≥2 distinct operators as the
   primary? 1 = monoculture flag → the next `/forge` must rotate.

Recommendation (advisory — redirects, never blocks): `CONTINUE` · `REFRAME` (core intact, auxiliaries
exhausted — re-forge) · `LATERAL` (region closed — re-prospect) · `STOP_AND_REPORT` (a decision only the
human can make).

## Write-through (tree FIRST, then narrative)
`tree_update_node` (status, scope grade + one-line why, the conversion output as insight); `tree_prune`
with reason for killed directions; then the run-log narrative. Update the forge backlog per the
conversion output.
