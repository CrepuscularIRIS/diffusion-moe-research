# The taste-operator palette — the fourth axis (`/forge` step 3‴)

> Frames fix *which mathematics*; operators fix *which object is modeled inside it*. An operator is a
> reusable **modeling-object-shift MOVE** (建模对象迁移): what the field models → what structure that omits
> → the more natural object. Operators are reverse-inferred from breakthroughs AND from our own `/autopsy`
> anomalies, then de-domained so they transfer. The pipeline HANDS the operator; the model derives.
> **Four axes is the ceiling** (schools · types · frames · operators) — a fifth is the menu disease at the
> meta level.

## The card schema
`operator_name` · `one_sentence_core_move` · `old_object_pattern` · `new_object_pattern` ·
`mathematical_frame` · **`core_simplification`** · **`differential_prediction`** · **`cheap_probe`** ·
`failure_signature` · `transfer_targets` · `positive_examples` · `when_it_misleads` · `source_episodes`.

The three bold fields are **load-bearing**: without `core_simplification` + `differential_prediction` +
`cheap_probe`, an operator is 术语, not taste. The differential must survive the **deletion test** (strip
the new vocabulary; if no prediction disappears, it is relabeling → KILL).

## Retrieval, not a menu
The bank lives as **project-side markdown** (sealed cards the deploying project maintains); this file
ships the machinery + two exemplars. `/forge` matches the problem's `failure_signature` to **1–3**
operators, forces **≥1** candidate from ONE, and **rotates** across rounds — never a full-bank sweep.
`/prospect` retrieves operators as a mining **prior** (a fit surfaces a candidate problem — it still needs
STAKES + SURPRISE).

## Accumulation (the bank grows only through the gate)
A new operator (usually an `/autopsy` `[OPERATOR-CANDIDATE]` — own anomalies are the only
off-distribution input we possess) enters ONLY via both: (1) the **corrosion gate** — three load-bearing
fields present, differential survives the deletion test; and (2) an **INDEPENDENT audit** — the one
invariant: the proposer cannot self-grant an operator into the bank. On FAIL it stays a scoped constraint.
Before an operator drives a real `/prereg`, the external brain deep-reads its `source_episodes` just-in-time
to verify the differential + probe are faithful to the actual method.

## Guardrail
An operator grants a candidate **no validity** — it still faces `/prereg` + `/adversary`. The bank expands
the PRIOR, not the evidence.

## Two exemplars (format reference)

### memory-kernel-closure
- **core_move:** when coarse-graining drops degrees of freedom, don't pretend they vanished — write their
  aggregate effect as an explicit memory kernel + noise on the kept variables.
- **core_simplification:** a tiny resolved state that stops paying the "my one-step model is silently
  wrong" tax.
- **differential_prediction:** a non-Markov lag structure — error a longer context does NOT fix but a
  kernel term does; a pure current-state model predicts none of it.
- **cheap_probe:** fit Markov baseline + a small learned kernel head; kill if kernel weight ≈ 0.

### transport-coupling-ization
- **core_move:** replace matching/similarity heuristics with a mass-flow coupling (incl. Gromov–Wasserstein
  across spaces without shared coordinates).
- **core_simplification:** one transport plan replaces pairwise-similarity heuristics + normalization patches.
- **differential_prediction:** a plan whose marginals + cost a soft-attention map cannot expose; strip
  "OT/coupling" and the marginal-feasibility claim disappears.
- **cheap_probe:** OT coupling vs attention on a cross-modal swap test, before any training.
