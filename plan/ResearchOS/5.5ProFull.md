下面是我建议的 **Research-OS v1.0 内容提炼版**。它不是 v0.10，不再加新机制，而是把系统压回“少数承重概念 + 清晰链路 + anti-accretion 约束”。目标文档的核心判断是：**protect the GENERATOR and the ONE INVARIANT, cut the ceremony**；也就是保留生成能力和安全边界，删除围绕它们增生出来的流程仪式。

---

# Research-OS v1.0 — 大道至简版

## 0. One Line

**Taste is a prior over expected-information-per-effort.**

研究不是先跑实验再用门槛筛选，而是在投入成本前，用 taste 判断：
**什么问题值得问，什么实验值得付费，什么观察会改变计划。**

Filter 只能减法，generator 才能创造价值。
因此 v1.0 只保护两件事：

```text
GENERATOR: 产生有价值的问题与候选方案
ONE INVARIANT: 防止提出者自授有利 verdict
```

---

## 1. 三个不可剪核心

### 1.1 The One Invariant

**A verdict that helps the proposer if gamed must never be granted by the proposer.**

也就是：

```text
提案者只能自授 DOWN verdict：
  KILL / DOWNGRADE / RESCOPE

CLAIM_STANDS 必须来自 independent substrate：
  新上下文模型 / Codex hook / Pro / 外部审查

真正的 contribution / paper promotion 终止于 human。
```

这是 Research-OS 的安全核心。所有 claim、promotion、external commitment 都围绕它。目标文档明确把它列为第一优先级，且称其为 non-negotiable safety core。

---

### 1.2 `/forge` — Value Engine

`/forge` 不能被当作 ceremony 删除。它是系统打破模型单一先验的主引擎。

输入一个问题，输出 3–5 个候选方案，每个候选必须经过强制多样化：

```text
rival-school candidate
non-incumbent-frame candidate
taste-operator candidate
DIFF-PREDICTION
```

核心规则：

```text
No differential prediction ⇒ relabeling ⇒ KILL.
```

也就是说，如果一个新方案不能说明它相比 incumbent 会预测到什么不同的可观察现象，它就不是 idea，只是换词。目标文档特别强调 `/forge` 的 forcing 是 essence，不是 ceremony，因为它负责 anti-monoculture。

---

### 1.3 Taste Model

`taste.md` 是哲学核心，不是参考装饰。

```text
taste = prior over expected-information-per-effort
```

它包含三条 v1.0 必须保留的判断：

```text
generative taste > evaluative taste
progress = surprise that changed the plan
zero surprise = process farming
```

评估型 taste 只能判断“这好不好”；生成型 taste 问的是“什么会让它变好”。Research-OS 的能力来自后者。目标文档也明确说，progress 的单位不是 token、checklist 或实验数量，而是 **SURPRISE — an observation that changed the plan**。

---

# 2. v1.0 核心逻辑链路

最终 Research-OS 只需要一条主链路：

```text
Taste
  ↓
Prospect
  ↓
Forge
  ↓
Preregister / Run
  ↓
Adversary
  ↓
Autopsy / Regenerate
```

展开成执行逻辑：

```text
1. Taste first
   判断什么问题值得付费。

2. Prospect
   从异常、survey、capability delta、benchmark critique、transplant 中找问题。

3. Forge
   对一个问题生成 3–5 个真正不同的候选方案。
   每个候选必须有 DIFF-PREDICTION。

4. Run
   在 claim-bearing run 前锁定：
     metric + split
     accept / kill rule
     seeds / one-variable change

   运行后必须：
     read numbers from artifacts
     verify intervention fired
     no artifact, no number

5. Adversary
   只在这里处理 CLAIM_STANDS。
   提案者不能自授有利 verdict。

6. Autopsy
   每个 null result 必须转化为：
     constraint
     new candidate
     region-close
```

这条链路保留 `/prospect`、`/forge`、`/adversary`、cheap-probe、artifact-fidelity、surprise accounting、type-first verification 等承重概念，同时删除 Layer-2 gate-stack 和 scheduled ceremony。目标文档中 “PROTECT / CUT-FOLD” 的建议也正是这个方向。

# 3. v1.0 Commands：从 11 个压到 5 个

## 3.1 保留为 command

### `/prospect`

作用：找值得研究的问题。

保留最小输出：

```text
{Q, TYPE, WHY-NOW, STAKES, PROBE, SURPRISE}
```

五个 mine：

```text
own-log anomalies first
survey-as-problem-mine
capability deltas
benchmark critique
transplants
```

`/prospect` 是 generator，不是 backlog manager。它的任务不是列一堆可做项目，而是找到能产生 surprise 的问题。

---

### `/forge`

作用：把一个问题转成多个高价值候选方案。

最小规则：

```text
Generate 3–5 candidates.

Each candidate must include:
  school
  type
  frame
  operator
  DIFF-PREDICTION
  cheap probe
```

保留 regeneration rule：

```text
Backlog regrows from failure.
Never enumerate the whole menu up front.
```

---

### `/prereg`

作用：只在 claim-bearing run 前冻结实验契约。

极简字段：

```text
metric + split
accept / kill rule
seeds
one-variable change
```

删除 10-field 长模板。
保留一句：

```text
Exploration is free; claims require prereg.
```

---

### `/adversary`

作用：唯一 claim-boundary gate。

最小检查：

```text
1. TYPE-scoped verification
2. make the opposing baseline win if possible
3. claim-evidence map
4. allowed claim envelope
5. one invariant
```

把 `/rigor` 中真正有用的东西折进这里：

```text
statistical unit
paired comparison
seed variance / claim scope
```

`/adversary` 不需要成为大型 tribunal。它只回答：

```text
这个 claim 是否能站住？
它是否由 independent substrate 支持？
提案者有没有自授有利 verdict？
```

---

### `/autopsy`

作用：把失败转成学习，而不是写墓志铭。

最小规则：

```text
Boring-first triage.
Mechanism-level why.
DOWN-only scope grade.
Every null becomes:
  constraint
  candidate
  region-close
```

保留 conversion law：

```text
A failed run must feed the generator.
```

---

## 3.2 删除为独立 command，但保留其核心

| 原命令                            | v1.0 处理                                                     |
| ------------------------------ | ----------------------------------------------------------- |
| `/exp-verify`                  | 折进 Run：artifact read + anti-no-op                           |
| `/rigor`                       | 折进 `/adversary`：unit + paired test + allowed claim envelope |
| `/compass`                     | 变成持续反射：surprise accounting + type drift                     |
| `/epistemic-calibration`       | 被 one invariant 覆盖                                          |
| `/irreversible-decision-audit` | 被 cheap-probe + human boundary 覆盖                           |
| `/artifact-acceptance`         | 只保留为外部交付前 handoff checklist                                 |

目标文档给出的 function map 也明确指出：`/rigor` 适合折叠，`/compass` 的 scheduled command ceremony 可以变成 continuous reflex，Layer-2 commands 的锐利部分已经被 one invariant 和 cheap-probe 覆盖。

# 4. v1.0 Keywords 词库

这些是必须保留的关键词。每个词都承重，不是装饰。

```text
taste-as-prior
generative > evaluative taste
surprise accounting
the one invariant
perspective-shift / 视角转换
omitted-structure / 第一性原理
object-shift / MOS / 建模对象迁移
the 4 axes
DIFF-PREDICTION
cheap-probe
kill-before-confirm
surprise-symmetric
artifact-fidelity
name-the-TYPE-first
anti-menu / regeneration
don't-circle / region-close
Pro generates · Opus operates · Codex checks · human sets taste
```

压缩定义如下：

| Keyword                         | v1.0 定义                                             |
| ------------------------------- | --------------------------------------------------- |
| `taste-as-prior`                | 在实验前判断什么值得付费；门槛不能替代 taste。                          |
| `generative > evaluative taste` | 目标不是筛掉坏想法，而是生成好想法。                                  |
| `surprise accounting`           | 进展 = 改变计划的观察 / 成本。                                  |
| `the one invariant`             | 提案者只能自授 DOWN；CLAIM_STANDS 需要 independent substrate。 |
| `perspective-shift`             | 从默认局部 checklist 跳到外部视角。                             |
| `omitted-structure`             | 找出默认建模隐藏掉的结构。                                       |
| `object-shift / MOS`            | 从旧对象迁移到更自然的新对象。                                     |
| `the 4 axes`                    | schools / types / frames / operators。               |
| `DIFF-PREDICTION`               | 新对象必须预测 incumbent 不能预测的可观察差异。                       |
| `cheap-probe`                   | 优先做每种结果都有信息的最便宜实验。                                  |
| `kill-before-confirm`           | 先找杀死方案的证据，再找确认方案的证据。                                |
| `artifact-fidelity`             | 数字必须从本 session artifact 读出，不能靠记忆。                   |
| `name-the-TYPE-first`           | 先确定研究类型，再决定验证方式。                                    |
| `anti-menu / regeneration`      | backlog 从失败中再生，不预先列满菜单。                             |
| `region-close`                  | 同一根因杀死 2–3 个机制后，关闭区域，横向迁移。                          |

这些词基本覆盖了目标文档中列出的 load-bearing keyword lexicon。

# 5. 五个 reference docs：保留，但降为 palette

v1.0 不需要把 reference docs 写进 command 正文里。它们应该保留为 generator palette：

```text
taste.md
schools.md
frames.md
taste-operators.md
research-types.md
```

定位：

```text
taste.md              philosophy
schools.md            how to think

```

这五个文档是真正给 `/prospect` 和 `/forge` 提供原料的部分，不应删除。目标文档也明确说它们是 generator raw material，应作为 optional palette 保留。

# 6. Arbor 的位置

Research-OS v1.0 不应该吞掉 Arbor execution engine。只保留 Arbor 作为事实账本和结构 substrate：

```text
idea-tree
eval_run
git_merge_branch
worktree isolation
generate_report
open_dashboard
```

核心价值：

```text
tree = canonical structure
eval_run = score from actual command output
worktree = execution isolation
merge = no-regression gate
```

尤其保留 `eval_run`，因为它防止“凭记忆写数字”的漂移。目标文档明确把 Arbor substrate/store 与 execution engine 分开，前者保留，后者对 methodology cut 来说是 orthogonal。

---

# 7. Anti-Accretion Rule

这是 v1.0 和 v0.10 的分界线。

每当系统出错，不允许第一反应就是加新 command、新 gate、新 checklist。必须按顺序问：

```text
1. Was it a bug?
2. Did the agent simply fail to follow the simple rule?
3. Can an existing piece absorb it?
4. Can we delete something?
5. Only then may we add a mechanism.
```

加机制必须满足：

```text
one-in-one-out
fixed parts-budget
adding requires deleting something else
```

目标文档特别要求安装 anti-accretion rule，因为之前系统已经出现过“删掉 gate-stack 后又重新膨胀”的问题。

# 8. v1.0 最终正文草案

下面这一版可以直接作为 `README.md` 或核心 `SKILL.md` 的浓缩正文。

```markdown
# Research-OS v1.0

Research-OS is a small research operating system for generating, testing,
and bounding research ideas.

Its core is not process. Its core is taste.

Taste is a prior over expected-information-per-effort.
A good system spends effort only where an observation can change the plan.

## The One Invariant

A verdict that helps the proposer if gamed must never be granted by the proposer.

The proposer may self-grant DOWN verdicts only:
KILL, DOWNGRADE, RESCOPE.

CLAIM_STANDS requires an independent substrate:
fresh context, external model, Codex hook, Pro, or human review.

Contribution-level promotion terminates in the human.

## The Generator

Research quality comes from generation, not filtering.

Use /prospect to find problems.
Use /forge to create 3–5 candidate approaches.

Each forged candidate must include:
- school
- type
- frame
- operator
- DIFF-PREDICTION
- cheap probe

No differential prediction means relabeling, not an idea.
Kill it.

A live programme does not enumerate a full menu up front.
It regenerates from failure.

## Run Discipline

Exploration is free.
Claims require preregistration.

Before a claim-bearing run, freeze:
- metric + split
- accept / kill rule
- seeds
- one-variable change

After the run:
- read numbers from artifacts
- verify the intervention fired
- never write numbers from memory

No artifact, no number.

## Claim Boundary

/adversary is the only claim gate.

Check:
- research TYPE
- opposing baseline
- claim-evidence map
- statistical unit
- paired comparison
- seed variance / claim scope
- worth

The proposer may not grant CLAIM_STANDS.

## Learning

Every null result must become one of:
- constraint
- new candidate
- region-close

If 2–3 mechanisms die from the same root cause,
the region is dead. Move laterally.

Progress is surprise that changed the plan.
Zero surprise means process farming.

## Palettes

The generator may draw from:
- taste.md
- schools.md
- frames.md
- taste-operators.md
- research-types.md

These are palettes, not gates.

## Arbor

Use Arbor as the substrate:
- idea-tree is the canonical structure
- eval_run records scores from actual outputs
- worktrees isolate execution
- merge protects against regression

## Anti-Accretion

When something breaks, do not add a mechanism first.

Ask in order:
1. Was it a bug?
2. Did the agent fail to follow the simple rule?
3. Can an existing piece absorb it?
4. Can we delete something?
5. Only then add a mechanism.

Adding requires deleting something else.
The parts budget is fixed.
```

# 9. 最终提炼结论

Research-OS v1.0 应该是：

```text
一个 taste-first 的研究生成系统，
一个 one-invariant 的 claim 安全系统，
一个 artifact-fidelity 的实验账本系统，
一个 failure-regenerates-backlog 的学习系统，
再加一个 anti-accretion 的自我收敛规则。
```

不应该是：

```text
更多 command
更多 gate
更多 checklist
更多 evidence ceremony
更多部署细节
更多解释性长文档
```

最终保留的本质是：

```text
Taste decides where to spend.
Prospect finds valuable questions.
Forge creates non-monoculture candidates.
Run reads artifacts, not memory.
Adversary bounds claims.
Autopsy converts failure into next search.
Anti-accretion keeps the system small.
```
