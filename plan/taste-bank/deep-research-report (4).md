# Robotics VLA World Model 与多模态 Grounding 的建模对象转移图谱

## 研究口径与判据

这份图谱不是按领域流水账列论文，而是把“**建模对象发生了什么转移**”作为唯一主轴来筛选。我的纳入标准是：一项工作必须把原本默认建模的对象，替换成另一个更可组合、更可压缩、或更可控制的对象；其贡献不能只是换 backbone、加模块、上新 benchmark、做大数据、或纯工程堆叠。这里的“旧对象→新对象”“压缩了什么复杂性”“新能力是什么”，很多时候并不是作者原文直接写出的字段，而是我根据论文机制做的**解释性归纳**。近年的综述也支持这种视角：机器人 world model 正在从任务专用 dynamics predictor，转向更一般的 predictive infrastructure；语义地图正从闭集标签向开放词汇、可查询、任务无关的表示演化；object-centric robotic manipulation 已经被明确看作一条独立路线；而 POMDP/Belief-space 传统则一直在把“状态估计”提升为“显式被规划的对象”。citeturn0search0turn0search3turn0search5turn23search16

本次我保留了 **53 个** 案例，尽量覆盖经典与长尾，并优先选择那些可迁移的底层“建模动作”：把像素换成 belief、把场景换成 object/graph/field、把连续动作换成 skill/plan/chunk、把 sim-to-real gap 换成 context/residual/distribution、把语言 grounding 换成 3D scene graph 或 language field。相反，很多只是在“统一成 token 序列”但没有显著改变可预测/可测/可控对象的工作，我没有放进主表，或只在边界部分提及。citeturn0search0turn0search3turn23search16

## 代表性分组分析

最反复出现的第一类动作，是把**“状态”从观测或单点估计，转成某种足够统计量**。Belief-space planning、Integrated TAMP in Belief Space、PSR、PEARL、VariBAD、PlaNet、Dreamer、TD-MPC，这些工作虽然技术细节差别很大，但都在做同一件事：承认“你真正要控制的不是传感器帧，也不是人造标签状态，而是一个对未来决策足够的内部对象”。有些工作把它做成 belief distribution，有些做成 latent stochastic state，有些做成 value-equivalent latent，有些做成 task context posterior。共同收益是：把观测噪声、历史依赖、任务不确定性和长时 credit assignment，从原输入空间中抽出来，塞进一个更可规划、更可过滤、更可复用的对象里。citeturn23search1turn23search14turn21search1turn7search7turn8search5turn1search0turn1search1turn20search4turn19search0

第二类动作，是把**场景从“整图/整帧”转成“对象、关系、场”**。Interaction Networks、VIN、OODP、FOCUS、Transporter、Where2Act、CLIPort、PerAct、NDF、VoxPoser、LERF、ConceptFusion、HOV-SG、MoMa-LLM、SayPlan 都属于这一簇。它们共享一个判断：机器人很多困难不是因为模型不够大，而是因为对象选错了。若把场景当作整图 embedding，你很难获得组合泛化、关系推理、空间检索、开放词汇 grounding；一旦换成 object graph、affordance field、3D value map、language field、hierarchical scene graph，机器人 suddenly 获得了旧对象难以直接支持的能力，例如：开放词汇查询、局部可作用部位发现、大场景子图检索、跨实例对齐、动作对应关系检测，以及基于场的约束组合。citeturn14search18turn14search3turn16search0turn20search1turn3search0turn3search1turn3search2turn4search0turn13search3turn3search3turn13search0turn13search1turn15search2turn15search1turn15search0

第三类动作，是把**动作从“瞬时控制量”转成 skill、plan、chunk、graph edit、residual 或 manifold policy**。Options、MAXQ、DMP、ProMP、Latent Plans from Play、Neural Task Graphs、RMPflow、Contact-Implicit TO、Logic-Geometric Programming、PDDLStream，都不是在“更好地输出一个 action vector”，而是在重新定义 action/plan 应该是什么。某些工作把它变成 temporally extended option；某些变成 attractor dynamics；某些变成 trajectory distribution；某些变成 latent plan；某些把 manipulation 重写成 graph edit；某些把解析控制之上再叠一个 residual policy。这类转移的核心收益，是将长时任务的组合复杂性，从时间轴直接搬运到结构对象上：skill library、subtask tree、trajectory distribution、contact mode、task skeleton、stream objects。citeturn6search0turn6search5turn25search2turn25search7turn1search3turn5search11turn9search12turn9search1turn9search2turn9search3turn7search0turn17search0

第四类动作，是把**sim-to-real gap 与可靠性问题，从“调不好”改成“显式对象”**。UP-OSI 把 gap 变成 online-identifiable dynamics context；Domain Randomization 把它变成 distribution coverage 问题；SimOpt 把它变成可以被真实数据更新的 simulator parameter distribution；Residual RL 则把不可建模部分集中到 residual policy。这个簇的共同风格非常重要：它们没有把“现实差距”当作工程 leftover，而是当作可建模、可估计、可优化的一等对象。citeturn8search0turn8search2turn7search1turn7search0

## 跨领域迁移地图

如果把这些案例当作“taste atlas”，最值得迁移到 VLA、MLLM、diffusion LLM 和 systems 的，不是某一篇论文的具体架构，而是几种反复出现的低层操作。

第一种是**beliefification**：把输入流变成带不确定性的内部状态。对未来的 VLA 或多模态 agent，这意味着不要只做 token-cache，而要做可更新 belief state；对 speculative decoding，这意味着可以把“草稿前缀”看成一类 uncertainty-bearing latent belief，而不是硬性的已定文本；对 dataset evaluation，这意味着评估数据是否足以恢复 task-sufficient state，而不是只看覆盖度。Belief-space planning、PSR、PEARL、VariBAD、PlaNet、Dreamer 都支持这一点。citeturn23search1turn21search1turn7search7turn8search5turn1search0turn1search1

第二种是**fieldization**：把动作目标或 grounding 目标建成场，而不是点。Transporter 的 correspondence field、Where2Act 的 affordance field、PerAct 的 voxel action field、VoxPoser 的 3D value map、LERF 的 language field，本质上都在说：一个任务往往不是“输出一个动作”，而是“在空间中定义一个连续可查询的目标地形”。这对多模态融合和 embodied MLLM 尤其重要，因为语言条件、视觉证据、几何约束可以自然叠加在同一个场对象中。citeturn3search0turn3search1turn4search0turn3search11turn13search0

第三种是**graphization**：把长程任务从序列重写为图搜索、图编辑、子图检索或 scene graph 更新。Neural Task Graphs、Sequential Manipulation Planning on Scene Graph、SayPlan、MoMa-LLM、HOV-SG 展示了图对象在组合任务、开放环境和动态 replanning 中的巨大优势。对于 systems 与 agent orchestration，这种迁移尤其自然，因为系统本来就更像约束图、依赖图、数据流图，而不是 flat sequence。citeturn5search11turn17search0turn15search0turn15search1turn15search2

第四种是**residualization**：把“主模型做不到的那部分”单独抽为 residual、correction、calibration object。Residual RL、SimOpt、PETS、PILCO 实际上给了一个非常实用的设计原则：不要让所有误差都挤在一个大模型里，把 gap、uncertainty、calibration、correction 明确化，经常更稳定、更省数据，也更可验证。这个思路对 systems、distillation、runtime alignment 都有迁移价值。citeturn7search0turn7search1turn18search0turn24search3

## 初步模式观察

从这 53 个案例里，可以先做几条**尚不终局化**的观察。

最明显的一条是：**建模对象越接近“可组合的 sufficiency object”，越容易形成长期影响力。** 这里的 sufficiency object 可能是 belief、latent state、object graph、affordance field、trajectory distribution、scene graph、residual policy；它们不一定是更真实的世界，却往往更接近“决策真正需要保留的信息”。MuZero 的 value-equivalent model、TD-MPC 的 task-oriented latent、SayCan 的 affordance-weighted skill distribution，都说明了这一点：不是所有可预测信息都值得建模，关键是对规划/控制/grounding 足够。citeturn19search0turn20search4turn5search6

第二条是：**很多突破并不是增加表达能力，而是删除不必要自由度。** DMP 删掉了对原始轨迹逐点追随的执念； ProMP 把示范多样性收进分布； PerAct 把 6-DoF 动作转成 3D detection； Transporter 把 pick-place 从动作回归改成 correspondence； HOV-SG 把 dense map 压成 hierarchy。它们共同显示，真正的“对象转移”常常是压缩自由度，而不是增加网络深度。citeturn25search2turn25search7turn4search0turn3search0turn15search2

第三条是：**开放词汇 grounding 的真正关键，不是更大的语言模型，而是更好的中间对象。** 仅靠文本/图像 token，通常难以长程执行；一旦引入 3D semantic map、language field、scene graph、3D value map，语言才真正获得可查询、可验证、可重规划的执行接口。SayPlan、MoMa-LLM、LERF、ConceptFusion、VoxPoser 都在不同层面验证了这一点。citeturn15search0turn15search1turn13search0turn13search1turn3search11

第四条是：**sim-to-real 成功案例几乎都把现实差异对象化了。** 不是遮住它，而是给它一个数学载体：parameter distribution、online context、residual、trajectory distribution、uncertainty ensemble。只要差异仍被当成“训练不够”或“测试太难”，方法通常不会形成可复制 operator。citeturn8search0turn8search2turn7search1turn7search0turn18search0

## 边界与不确定性

这份图谱里有少数**边界型**案例，最典型是 RT-2。它当然很有影响，但其核心对象转移更接近“把动作序列纳入与语言共享的 token/object space”，而不是像 belief、affordance、object-centric state 那样重新定义环境内部对象。之所以仍然保留，是因为它确实把“机器人动作”从连续控制变量转成了可与 web-scale语义共训练的对象，因此在 VLA 发展史中具有方法论意义。citeturn12search2turn12search6

相反，我刻意没有把很多“更大更强的统一 VLA”放进主表。原因不是它们不重要，而是若其主要贡献只是更大数据、更统一序列化接口、更强 backbone，而没有清楚地完成“旧对象→新对象”的替换，那么它们更适合被归入 scaling/engineering 史，而不是“建模对象转移”史。这也是为什么主表里 object graph、scene graph、belief、residual、latent plan、affordance field 的密度明显高于纯 token 化系统。这个取向与近年的 world model、semantic mapping、object-centric manipulation 综述描述出的主线大体一致。citeturn0search0turn0search3turn0search5

从这些案例中，初步能看到几种高频“建模对象转移模式”：**状态充分统计化**、**对象关系化**、**空间场化**、**动作时程化**、**任务图化**、**现实差异残差化**、**语言 grounding 的 3D 化**。这些还不该被过早上升为最终理论，但已经足够作为“Modeling Taste Atlas”的一组初始 taste/operator：以后无论你读 VLA、MLLM、diffusion world model、speculative decoding、OOD evaluation 还是 systems paper，都可以先问：它到底有没有换建模对象？换成了什么对象？那个对象删掉了什么复杂性，又新增了什么可预测、可测量、可控制的能力。
