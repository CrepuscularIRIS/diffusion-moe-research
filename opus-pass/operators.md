### NET-NEW operators promoted from the Pro pass (absent from the ★ set below)

#### N1. `recoverability-gated-fusion` ★★  (our own VLA anomaly — the CAF-VLA operator)
- **one_sentence_core_move:** before reweighting modalities, TEST whether any modality is recoverable from
  the others; treat reliability as a routing object ONLY where redundancy actually exists.
- **old→new object:** always-fuse / always-reweight modality scores → a recoverability matrix + a reliability
  policy over redundant / conflicting / indispensable evidence.
- **mathematical_frame:** conditional-redundancy graph / information-recovery matrix / causal-graph (do-gate).
- **core_simplification:** removes impossible rescue work — if every modality is individually necessary and
  none is reconstructible from the rest, fusion-reweighting has no redundant evidence to exploit.
- **differential_prediction:** reliability-reweighting helps under benign conflict / missing-noisy single
  modality, but CANNOT help when recoverable_fraction≈0 and every removal collapses success; forced
  reweighting then does nothing or degrades. Deletion test: strip "recoverability" and the "reweighting is
  futile here" prediction disappears.
- **cheap_probe:** leave-one-modality-out matrix + a tiny conditional imputer BEFORE training; kill
  fusion-reweighting if all removals collapse success AND all conditional-recovery probes fail.
- **failure_signature:** all single-modality ablations fatal; pairwise substitution never restores success;
  reliability weights saturate but success doesn't recover.
- **transfer_targets:** VLA fusion (precondition check before any reliability router — the CAF-VLA screen) ·
  MLLM grounding (irreducible modality conjunction) · OOD (separate redundancy- from contradiction-failures)
  · dataset eval (label non-recoverable multimodal tasks).
- **source_episodes:** OUR VLA `recoverable_fraction≈0` anomaly (libero_goal, N≥48pp, G<0, 4 suites) · DCR/ADA
  conflict arbitration · EBMC reliability/trust distillation · C2MF context-credibility.
- **generation_test:** the CAF-VLA Rank-1 recoverability screen — the operator says the seed failure does NOT
  mean "build a better router"; it means "test recoverability first, and if absent, reach for a conjunctive
  object (belief / factor-graph / object-slot / affordance field), not a reweighting object."

#### N2. `barrier-certificate-localization` ★  (VERIFIED — Control Barrier Functions, IEEE 7782377)
- **core_move:** replace a soft penalty over future trajectories with a LOCAL certificate that forbids leaving
  the safe set (control barrier function / forward-invariant set / local QP filter).
- **core_simplification:** no need to search the whole future to certify safety — the hard object is a local
  inequality at the boundary (the state derivative).
- **differential_prediction:** the barrier rejects unsafe actions even when the policy/reward ranks them high,
  and rejections correlate with boundary-crossing risk; a penalty term with no forward-invariance test does not.
- **cheap_probe:** on logged trajectories, define a candidate barrier and test whether violations predict
  future failure BETTER than reward or confidence.
- **failure_signature:** rare-but-catastrophic safety failures; average reward improves while boundary
  violations persist. **transfer:** VLA action safety filter · MLLM geometric no-go · systems SLA barrier ·
  spec-decoding schema-validity barrier before token acceptance.

#### N3. `dependency-bounded-assembly` ★  (VERIFIED — dependency-aware parallel decoding, arXiv 2603.12996)
- **core_move:** replace independent parallel commits with an explicit dependency graph over candidate pieces;
  assemble only LOW-coupling sets (conditional-independence test / attention-derived coupling / conflict graph).
- **core_simplification:** kills the false factorization assumption — the expensive object is no longer the
  full joint, only the dependency boundary among candidate commits.
- **differential_prediction:** high-confidence but strongly mutually-dependent tokens FAIL when committed
  together, while lower-confidence conditionally-independent sets commit safely in parallel; confidence alone
  can't tell them apart.
- **cheap_probe:** estimate pairwise dependency (attention / conditional entropy / remask sensitivity) on
  frozen samples; compare confidence-only vs dependency-pruned commit sets on denoising loss + exact-match.
- **failure_signature:** parallel decoding gains speed but loses correctness; confidence high yet simultaneous
  commits break syntax/math/code. **transfer:** diffusion-LLM joint assembly · spec-decoding multi-token accept
  · systems (batch only low-coupling jobs).

#### N4. `two-part-code-ization` ★★  (MDL — Grünwald tutorial + Hinton–van Camp + Kolmogorov; the bank's biggest gap)
- **one_sentence_core_move:** stop optimizing/reporting FIT; model the TOTAL MESSAGE — bits(model) + bits(residuals|model) — as one objective in one currency, so complexity control is internal, not an external val-tuned penalty.
- **old→new object:** training loss/likelihood with complexity handled externally (val-tuned regularizers, param counts, early stopping) → a two-part code L(H)+L(D|H); the model is a message you must pay for; idealized = Kolmogorov structure function (structure bits + incompressible residual).
- **mathematical_frame:** algorithmic information / coding theory (two-part & universal/NML codes; Kraft ⇒ codelengths=probabilities).
- **core_simplification:** removes the external overfitting apparatus — plain weight-decay is the degenerate special case (two fixed Gaussians), exposing what the tuned coefficient was secretly pricing (σ_j²/σ_w²).
- **differential_prediction:** (selection facet) two models with IDENTICAL train loss have ORDERED held-out performance, predicted by the MODEL-bits term measured on training data ALONE — pure likelihood and #29 IB (which needs a target Y) have no signal to order them; equal-param-count families of different functional form get different penalties. (trajectory facet = `structure-noise-bipartition`, Coffee Automaton) split the code into structured/coarse-grained bits vs incompressible residual and instrument BOTH along a process trajectory: the structure channel is NON-MONOTONE (peaks mid-process) while entropy is MONOTONE, and peak-structure scales with interface (∝n) while max-entropy scales with volume (∝n²); a single scalar loss predicts neither. Deletion test: strip "code length of the model" and both the equal-loss ordering AND the peak vanish. SURVIVES.
- **cheap_probe:** prequential codelength on a FROZEN model (sum −log p of each next chunk before conditioning — ICL is the online learner, no gradient); effective-bits = the bit-width at which held-out loss degrades; **residual-compressibility** (gzip/LM-compress a fitted model's ERRORS — if residuals still compress, structure remains and total codelength was not minimized, *even if likelihood looks good*). Trajectory probe: structure=gzip(coarse-grained state) vs entropy=gzip(raw) across checkpoints/denoising-steps — MANDATORY controls: finer-threshold artifact control + a decoupled negative control (a naive coarse-graining manufactures a FAKE peak).
- **failure_signature:** equal-train-loss candidates must be ranked without a val split; regularization tuning is circular; comparing model classes of different form; a benchmark gain may be complexity-purchased; "representations got interesting then collapsed" mid-training.
- **transfer_targets:** dataset-eval (prequential codelength of labels|inputs → noisy labels = incompressible, leaked/dup data = anomalously short) · distillation (distill iff bits(student)+bits(correction) < bits(teacher-message); distill the structure-PEAK checkpoint not the final one) · diffusion-LLM (structure peaks at intermediate noise → pick remask schedule by peak, not loss) · OOD · the /forge NECESSITY GATE re-cast as bit-accounting.
- **generation_test (VLA 刷分):** the necessity gate becomes bit-accounting — a robustness adapter is NECESSARY only if bits(adapter)+bits(residual-errors) < bits(plain-LoRA message) at equal robustness; the residual-compressibility probe KILLS adapters whose Δ is complexity-purchased BEFORE the claim run.
- **dedup defense:** NOT #29 information-bottleneck (rate–RELEVANCE, needs a target Y — this prices the MODEL+residual message with no target); NOT #21 sparse-support. `structure-noise-bipartition` folded here (same object, distinct differential).
- **source_paper:** `.A Tutorial Introduction to the MDL Principle` (Grünwald §1.3–1.4,2.5) + `Keeping NNs Simple by Minimizing the Description Length of the Weights` (Hinton–van Camp) + `kolmbook` (K(x) substrate); trajectory facet = Aaronson/Carroll/Ouellette, *Coffee Automaton*, arXiv:1405.6903.

#### N5. `bits-back-refund-pricing` ★  (Hinton–van Camp bits-back + Variational Lossy Autoencoder)
- **one_sentence_core_move:** price any stochastic component (noisy weight, latent code) not at raw encoding cost but at KL(posterior‖prior) — the noise entropy is a REFUND, so information migrates to whichever channel is cheapest under this price.
- **old→new object:** deterministic params/latents priced at raw precision; posterior-collapse seen as an optimization bug; KL seen as a generic regularizer → a communication protocol where true cost = C−R = KL(Q‖P); ELBO = expected codelength; each piece of info sits in its cheapest channel.
- **mathematical_frame:** variational coding / Helmholtz free energy.
- **core_simplification:** removes the paradox that noise is expensive (high-variance components become CHEAP) and the trial-and-error "why is my latent dead" debugging — placement is predicted from coding cost alone.
- **differential_prediction:** (i) noisy weights cost LESS than precise ones (naive two-part coding predicts more), and generalization tracks ΣKL not param count; (ii) with a decoder locally able to model the data the latent is ignored AT THE OPTIMUM — better optimizers will NOT fix it; the collapse cost is exactly KL(q(z|x)‖p(z|x)); (iii) shrinking the decoder's receptive field forces a SPECIFIC, measurable migration of global info into the latent (VLAE: 13.3 nats lossy vs 37.3 full). Deletion test: strip the refund vocabulary and (i) inverts, (ii) disappears, (iii) loses its bit-accounting. SURVIVES.
- **cheap_probe:** noise-tolerance = effective bits (add weight noise on a FROZEN model until held-out loss degrades); per-dimension KL(q‖p) reads bits actually purchased → diagnoses dead dims/collapse with NO training; a miniature receptive-field sweep verifies the predicted KL rise + content migration in minutes.
- **failure_signature:** a latent/side channel is ignored despite ample capacity; you must decide WHERE information should live among competing channels (latent vs decoder, summary vs local window, plan vs policy).
- **transfer_targets:** long-context (KV-summary vs local-attention = two channels; constrain the local one to force global state into the summary) · VLA/MLLM latent plans · distillation (price student stochastic states at KL) · diffusion-LLM.
- **generation_test (VLA):** a latent-plan / intermediate-representation adapter atop the FROZEN Qwen2.5-VL action decoder WILL collapse at optimum unless the decoder's local scope is cut — the per-dim-KL probe kills such designs BEFORE training (a pre-training kill-gate for any latent-plan robustness candidate).
- **dedup defense:** NOT #28 latent-variable-ization (VLAE's latent machinery IS #28; the net-new is the KL-refund PRICING + channel-placement move); weight-noise WITHOUT the accounting = a killed reg-knob.
- **source_paper:** `Keeping NNs Simple…` §5–7 + `Variational Lossy Autoencoder` §2.2–3.2.

#### N6. `program-store-factorization` ★  (Neural Turing Machines — opens a new bank axis)
- **one_sentence_core_move:** factor a learned system into a small size-invariant PROGRAM (weights = data-independent access rule) × an unbounded addressable NON-parametric STORE (instance data), so capacity leaves the learning problem.
- **old→new object:** monolithic state/weights entangling rule and data (capacity ∝ params) → a controller emitting content/position-addressed read/write kernels over an external store whose capacity is a free, test-time-extensible resource.
- **mathematical_frame:** probability kernels over a discrete index set (content-similarity softmax + shift) coupled to a small controller dynamical system.
- **core_simplification:** weights encode only HOW to store/fetch, not WHAT is stored — the hypothesis class shrinks from "functions of the whole history" to "size-invariant access policies," which is why the learned solution is a compact algorithm at all.
- **differential_prediction:** (i) size extrapolation — trained ≤n, executes ≫n (NTM copy trained ≤20 runs at 120; matched LSTM collapses past 20); (ii) test-time capacity scaling — enlarge the store, extend competence with ZERO weight change (impossible for a param-entangled model); (iii) error signature = discrete local SLIPS (one duplicated/shifted vector), not global blur/decay.
- **cheap_probe:** rule/instance separability check (can a human write size-invariant pseudocode over pointer+array?); train the incumbent small (≤n), test at 3–6×n, look for a cliff at exactly n; hand the incumbent a trivial external scratchpad/retrieval cache at TEST time ONLY — if accuracy moves, storage (not rule) was binding.
- **failure_signature:** accuracy cliff exactly at the training-size boundary; difficulty scales with how much must be REMEMBERED, not rule complexity; adding params shifts the cliff but never removes it.
- **transfer_targets:** long-context (store-extension/retrieval beats window-growth exactly on rule-constant separable tasks, NOT holistic ones) · VLA (variable object count: fixed policy over an object registry vs re-encoding a scene into one vector) · speculative-decoding (draft policy as a program over the accepted-prefix store) · distillation (distill the access PROGRAM → student inherits length extrapolation).
- **generation_test (VLA):** under Layout/object-count perturbation a fixed policy over an addressable object registry should count-extrapolate where a scene-to-one-vector encoder cliffs — cheap to probe (train ≤k objects, test >k).
- **dedup defense:** OPPOSITE axis to #1 memory-kernel-closure / #25 belief / #36 sufficient-summary (those compress history into a fixed statistic and DISCARD data; this RETAINS raw data non-parametrically and learns only access) — new axis = **parametric-compression vs indexed-retention**. "bigger store" ≠ "bigger window" (killed).
- **source_paper:** Graves, Wayne & Danihelka, *Neural Turing Machines*, arXiv:1410.5401.

#### N7. `selection-output-ization` ★  (Pointer Networks)
- **one_sentence_core_move:** re-type the OUTPUT space from a frozen closed vocabulary to an INDEX over the instance's own input elements, so the feasible set is constructed per-instance and feasibility holds by construction.
- **old→new object:** softmax over a design-time-frozen inventory (fixed cardinality; one model per problem size) → a conditional distribution over the positions/elements of the CURRENT input (cardinality = instance size; the attention weights ARE the output).
- **mathematical_frame:** a per-instance output measure on the input's index set.
- **core_simplification:** deletes the closed-world inventory and the content-synthesis that can fabricate outputs — the model only RANKS given evidence; feasibility (output ∈ input) is by construction, not learned; also deletes retrain-per-size.
- **differential_prediction:** (i) one model solves variable and UNSEEN output cardinality (Ptr-Net trained n≤50 generalizes above it; a fixed-softmax seq2seq provably cannot even REPRESENT the task); (ii) ZERO out-of-feasible-set outputs, vs the incumbent's hallucinated/blurry direct-content predictions. Deletion test: strip "selection/index" and both vanish. SURVIVES.
- **cheap_probe:** compute the COPY RATE (fraction of target elements literally in the input) + whether the label space varies per instance; if copy-rate high, bolt a tiny pointer head on a FROZEN encoder and verify feasibility-violations→0 and size-extrapolation improves.
- **failure_signature:** hallucinated outputs outside the instance's legal set; label space padded to max-size with masking; retrain needed when instance size changes; targets that are verbatim input elements.
- **transfer_targets:** MLLM (extractive grounding: answer = region/token index → kills hallucination on extractive questions specifically) · VLA (grasp-target = point into the detected-object set → zero grasps at non-existent objects) · long-context (answer-by-citation vs regenerate) · speculative-decoding (verification as selection over draft candidates) · dataset-eval (split benchmarks by copy-rate).
- **generation_test (VLA/MLLM robustness):** under object-reference / language shift, a selection head (index into detected objects) predicts zero references to ABSENT objects + count-extrapolation — a fixed-vocabulary head predicts neither; cheap to probe via copy-rate + a frozen-encoder pointer head.
- **dedup defense:** NOT "add attention" (killed) — the move is RE-TYPING the output space (reusing attention AS the output distribution); NOT #26 distribution-output-ization (point→distribution over a FIXED space).
- **source_paper:** Vinyals, Fortunato & Jaitly, *Pointer Networks*, arXiv:1506.03134.

#### N8. `performance-law-ization`  (Scaling Laws — META: models the resource→performance law; overlaps launch-arithmetic §4.9)
- **one_sentence_core_move:** stop studying the model-at-a-configuration; the OBJECT is the fitted low-dimensional LAW mapping resources→performance (power-law terms + irreducible floor), and every decision (go/no-go, allocation, stopping) is read off the law's extrapolations, never off a full-cost run.
- **old→new object:** empirical config-by-config evaluation (tune shape, train to convergence, see what you get) → the resource–performance SURFACE L(N,D,C,S) as a parametric object fitted on cheap small runs, whose out-of-range extrapolation + constrained-budget optimum ARE the deliverable.
- **mathematical_frame:** power-law/critical-exponent asymptotics + additive bottleneck decomposition + Lagrange allocation on the fitted surface.
- **core_simplification:** collapses millions of architecture/HP DOFs into ~3 scalars (loss varies only % across 40× aspect-ratio once non-embedding N is fixed) and demotes the expensive run to a held-out TEST POINT of a cheap fit.

## Family 1 — Coarse-graining & closure (long-tail crown jewels: what to do with the DOFs you drop)

### 1. memory-kernel-closure ★
- **one_sentence_core_move:** when you coarse-grain and throw away degrees of freedom, don't pretend they vanished — rewrite their aggregate effect as an explicit memory kernel + orthogonal noise term on the variables you kept.
- **old_object_pattern:** a Markov / current-state model that assumes the discarded history is irrelevant (truncate context, single-step state).
- **new_object_pattern:** resolved variables + a history-dependent memory kernel + a structured noise/forcing object.
- **mathematical_frame:** Dynamical systems & control (projection-operator / generalized Langevin equation).
- **core_simplification:** you keep a tiny resolved state yet stop paying the "my one-step model is silently wrong" tax — the un-modeled past is given ONE dedicated object instead of leaking into every prediction as bias.
- **differential_prediction:** predicts a *non-Markov lag structure* — error that a longer context window does NOT fix but a memory-kernel term does; and a rare "forcing precursor" observable before history-dependent failures. A pure current-state model predicts none of this.
- **cheap_probe:** teacher-forced diagnostic — fit a Markov baseline, then add a small learned memory-kernel / predictive-state head; if kernel weight ≈ 0 or long-rollout error is unchanged, kill.
- **failure_signature:** reach for it when longer context helps sub-linearly, rollouts drift, or failures correlate with something that happened many steps ago.
- **transfer_targets:** diffusion-LLM (the measured ~3-nats sequential bias = a memory kernel the mask-CE model omits), speculative-decoding (beyond-Markov draft head — the DSpark goal), long-context LLM, VLA latent memory, systems backlog.
- **generation_test (diffusion-LLM / spec-decode):** *candidate* — the ~3-nats non-Markov bias in a dLLM is a Mori–Zwanzig memory kernel; add a low-rank kernel head reading the last-k committed tokens' hidden states. *Differential:* accepted-length/prefix-NLL improves ONLY where the token depends on lag>1, and the gain vanishes if you shuffle the k-window (negative control). *Cheap probe:* correlate per-position gain with an autocorrelation-lag estimate before any training. Kill if gain is lag-independent.
- **positive_examples:** Mori–Zwanzig formalism, generalized Langevin closure, Predictive State Representations, S4/SSM as a learned kernel.
- **negative_examples / when_it_misleads:** merely widening the context window is NOT this (no explicit kernel object, no lag-structure prediction); any RNN hidden state without a memory-object interpretation is relabeling.
- **source_episodes:** Mori-Zwanzig Projection (科学计算 #1, report-3), PSR (report-4), S4 (AI案例 B), own-anomaly "~3 nats sequential bias" (project autopsy ledger).

### 2. observable-lifting ★
- **one_sentence_core_move:** trade a low-dimensional nonlinear dynamics for a higher-dimensional space of *observables* in which the same evolution is (nearly) linear, so control and multi-step prediction become spectral algebra.
- **old_object_pattern:** integrate/autoregress the nonlinear state one hard step at a time.
- **new_object_pattern:** a lifted observable vector evolving under a (near-)linear operator; its eigen/slow-subspace.
- **mathematical_frame:** Spectral / operator (Koopman operator, EDMD).
- **core_simplification:** removes step-by-step nonlinear rollout — a single linear propagator gives many steps at once, and stability/controllability become eigenvalue questions.
- **differential_prediction:** predicts a *block-linear multi-step forecast* (k steps in one linear map) and a dominant-timescale / controllable-subspace decomposition that a one-step nonlinear predictor cannot expose. Deletion test: strip "Koopman/lifting" and the block-forecast + spectral-timescale claims disappear.
- **cheap_probe:** EDMD with a small learned dictionary on a toy system; test whether one fixed linear matrix predicts k-step-ahead as well as a nonlinear stepper, and whether eigenvalues match known timescales.
- **failure_signature:** reach for it when you need cheap long-horizon prediction, a linear controllable handle, or an interpretable timescale from a system you currently roll out step-wise.
- **transfer_targets:** speculative-decoding (a block-linear hidden-state predictor drafts k tokens at once), VLA world-model planning in a lifted latent, MLLM video dynamics, OOD slow-variable extraction.
- **generation_test (spec-decode):** *candidate* — learn a Koopman lift of the target LM's hidden state so one linear operator forecasts the next k hidden states, decode all k as a draft block. *Differential:* accept-rate of the linear-forecast block exceeds a same-size MLP draft head at equal params, and degrades gracefully with k (a spectral-radius prediction). *Cheap probe:* fit EDMD on frozen hidden-state traces, measure k-step reconstruction vs an MLP before training a head. Kill if linear ≈ MLP.
- **positive_examples:** Koopman operator, Carleman linearization, EDMD, DMD, HAVOK (chaos = linear + intermittent forcing), Koopman-MPC.
- **negative_examples / when_it_misleads:** a plain linear probe or PCA is not observable-lifting (no invariant-subspace / long-horizon claim); calling a wider net "lifted" without a linear propagator is relabeling.
- **source_episodes:** Koopman / Carleman / Perron-Frobenius / Takens / Lax pairs (科学计算 #2-6), DMD/EDMD/HAVOK/Koopman-MPC (report-3).

### 3. multiscale-correction-hierarchy ★
- **one_sentence_core_move:** stop solving on one resolution — represent the *error* on a hierarchy of scales and alternate cheap coarse global corrections with fine local ones (restrict → correct → prolong).
- **old_object_pattern:** a single-resolution solver/predictor that grinds down all error modes at one level.
- **new_object_pattern:** a coarse↔fine error/residual hierarchy with restriction and prolongation operators.
- **mathematical_frame:** Spectral / operator (multigrid error-frequency decomposition).
- **core_simplification:** decouples "high-frequency error is cheap to kill locally, low-frequency error is cheap to kill coarsely" — you never fight a stiff low-frequency mode at fine resolution.
- **differential_prediction:** predicts *resolution-independent convergence* (iterations to tolerance flat as problem size grows) and identifies WHICH error-frequency band a one-level method is stuck on. A single-level method predicts size-growing iteration counts.
- **cheap_probe:** two-level V-cycle vs single-level iteration on a toy; measure error-band reduction per pass and iteration-count vs size.
- **failure_signature:** reach for it when a coarse cheap pass and a fine expensive pass disagree, or when refinement helps only locally while a global bias persists.
- **transfer_targets:** speculative-decoding (small draft = coarse grid, target verify = fine correction, i.e. a V-cycle over decoding), coarse-to-fine multimodal fusion, hierarchical planning, dataset condensation (algebraic coarse variables).
- **generation_test (spec-decode):** *candidate* — treat draft/verify as one multigrid V-cycle: a coarse draft proposes a block, the target corrects only the low-confidence (low-frequency-error) positions, then a second coarse pass re-drafts the tail. *Differential:* wall-clock-to-accepted-token stays flat as sequence length grows (resolution-independence), unlike flat draft-then-verify. *Cheap probe:* measure whether target corrections cluster in a stable low-confidence band (restrictable) on logged runs. Kill if corrections are uniformly spread (no frequency structure to exploit).
- **positive_examples:** Multigrid, Algebraic Multigrid (coarse variables inferred from the operator itself).
- **negative_examples / when_it_misleads:** any two-stage "small model then big model" without a restriction/prolongation error object and a resolution-independence claim is just cascading, not this.
- **source_episodes:** Multigrid, Algebraic Multigrid (report-3, 几何与离散化 / 动力系统).

### 4. collective-variable-ization
- **one_sentence_core_move:** replace the full high-dimensional trajectory with a few slow reaction-coordinates / metastable states and do the modeling, exploration, or biasing on THAT low-dim object.
- **old_object_pattern:** the raw full-coordinate time series or a PCA/cluster compression that ignores dynamics.
- **new_object_pattern:** collective variables / a free-energy surface / a metastable-state transition graph.
- **mathematical_frame:** Statistical mechanics / mean-field (free-energy surface, order parameter).
- **core_simplification:** rare transitions and long timescales are peeled out of the raw coordinates, so you plan/explore over a handful of slow modes instead of the whole state.
- **differential_prediction:** predicts *timescale separation, transition bottlenecks, and rare-event trigger coordinates* — a dynamics-blind compression (PCA/UMAP/k-means) predicts none of these and its axes won't reproduce the true slow modes.
- **cheap_probe:** diffusion-maps / VAMP / MSM on a small simulator; test slow-mode reproducibility across seeds and whether biasing along the CV accelerates a known transition.
- **failure_signature:** reach for it when behavior is dominated by a few slow modes hidden in a fast high-dim signal, or when exploration keeps re-visiting the same basin.
- **transfer_targets:** VLA skill discovery (slow "skill coordinates" from play), LLM-runtime metastable failure states, dataset-eval coverage over slow modes, OOD precursors.
- **positive_examples:** Metadynamics, Diffusion Maps, Markov State Models, VAMP/VAMPnets.
- **negative_examples / when_it_misleads:** a PCA/UMAP plot is not a CV object unless it carries a timescale / rare-event differential; "slow feature" with no transition-prediction is decoration.
- **source_episodes:** Metadynamics, Diffusion Maps, MSM, VAMP/VAMPnets (report-3), Onsager-Machlup / Kramers rate (科学计算 #30-31).

## Family 2 — Local-interaction & conservation (long-tail: make a global coupling locally computable)

### 5. flux-conservation-ization ★
- **one_sentence_core_move:** stop modeling node values or pairwise similarity; model the conserved *flux* crossing local boundaries (a source→sink ledger) so what leaves A must enter B.
- **old_object_pattern:** node/point values, static adjacency, or attention similarity that need not conserve anything.
- **new_object_pattern:** cell flux / numerical flux / flow-token with an enforced continuity (conservation) constraint.
- **mathematical_frame:** Dynamical systems & control (finite-volume flux / continuity equation).
- **core_simplification:** conservation and wave/shock propagation stop being something a global state must memorize — they are carried automatically by local boundary fluxes.
- **differential_prediction:** predicts *shock/bottleneck location and congestion transfer* and forbids mass creation; an attention/similarity model predicts smooth correlations that routinely violate the ledger. Deletion test: drop "flux/conservation" and the balance-violation and bottleneck-location predictions vanish.
- **cheap_probe:** on a toy transport simulator, compare an adjacency/attention model vs a flux model on conservation error and whether a local intervention is traceable to a downstream bottleneck.
- **failure_signature:** reach for it when a model captures correlations but violates a physical/resource budget, or when you need to localize where a quantity accumulates.
- **transfer_targets:** speculative-decoding & serving (KV / compute budget as a conserved flux over positions), systems queue routing, VLA contact-force flux, dataset-shift as mass transport.
- **generation_test (spec-decode / serving):** *candidate* — model the KV-cache / token-budget of a batched decoder as a conserved flux graph rather than a load vector, routing draft compute where "arrival minus service" accumulates. *Differential:* predicted stall positions coincide with measured KV-pressure spikes and the ledger closes (in=out) per step, which a load-vector scheduler does not guarantee. *Cheap probe:* on logged serving traces, test whether a conservation residual predicts stalls better than raw utilization. Kill if the residual is uninformative.
- **positive_examples:** Godunov / finite-volume, Lattice Boltzmann, Spatio-Temporal FlowNet, GFlowNet (flow on a DAG).
- **negative_examples / when_it_misleads:** renaming an attention normalizer "flow" with no conservation constraint is a killed transplant; a GNN edge weight is not a flux.
- **source_episodes:** Godunov/FV, Lattice Boltzmann (report-3), FlowNet (机器人 / AI案例), GFlowNet (report-2).

### 6. factor-graph-ization ★
- **one_sentence_core_move:** replace a global joint function / global constraint solve with local factors exchanging messages on a bipartite graph.
- **old_object_pattern:** a dense global objective, a monolithic constraint solve, or a whole-code check.
- **new_object_pattern:** a factor graph (variables + factors) with sum-product / belief-propagation messages.
- **mathematical_frame:** Rate-distortion / coding (Tanner / factor graph, message passing).
- **core_simplification:** exponential global coupling collapses to local marginalization; each constraint becomes a swappable module rather than part of one entangled solve.
- **differential_prediction:** predicts *local marginals, the specific factor a conflict originates from, and how swapping one factor changes the posterior* — a dense solver gives an answer but cannot localize the responsible constraint or accept a modular swap without full re-solve.
- **cheap_probe:** hand-build a small factor graph vs a dense solver on a toy; check marginal accuracy and whether a single-factor intervention is diagnosable.
- **failure_signature:** reach for it when a global solve is expensive/opaque, constraints are modular, or you need to attribute a failure to one sub-constraint.
- **transfer_targets:** speculative-decoding verification (draft/verify as extrinsic message passing between two decoders, à la Turbo), multi-agent / tool-chain consistency checks, multimodal grounding constraints, systems dependency graphs.
- **generation_test (spec-decode):** *candidate* — run draft and target as two constituent decoders exchanging *extrinsic* per-token soft information (Turbo-style) instead of a hard accept/reject, iterating to agreement on a block. *Differential:* iterating extrinsic messages raises accepted length over one-shot accept/reject at equal target-calls, and disagreement localizes to specific token factors. *Cheap probe:* on logged draft/target logits, test whether one message-passing round flips borderline rejects to accepts. Kill if no borderline mass exists.
- **positive_examples:** Factor graph + sum-product, Tanner graph, LDPC, Turbo codes, belief propagation.
- **negative_examples / when_it_misleads:** a generic GNN is not factor-graph-ization unless nodes carry explicit factor semantics; adding an adjacency feature is decoration.
- **source_episodes:** Factor Graphs / Tanner / LDPC / Turbo (report-3), Polar/LDPC (科学计算 #33-34), Physics-Based RL logic (AI案例 C).

### 7. far-interaction-compression
- **one_sentence_core_move:** don't compute every pairwise interaction or fill the whole volume — summarize the *distant / bulk* part as a low-rank hierarchical expansion (multipole) or a boundary-only integral, and keep full resolution only nearby.
- **old_object_pattern:** dense all-pairs interaction (O(N²)) or a volume-filling mesh over the whole domain.
- **new_object_pattern:** a hierarchical far-field / multipole summary + a boundary or near-field local term.
- **mathematical_frame:** Spectral / operator (hierarchical low-rank far-field, Green's-function boundary integral).
- **core_simplification:** the far/interior part is provably low-rank, so its cost drops from O(N²)/O(N³) to near-linear without losing the near-field detail that matters.
- **differential_prediction:** predicts that far interactions are *low-rank-summarizable to bounded error* — accuracy holds as you coarsen distant blocks, and cost scales ~O(N). A dense model predicts uniform sensitivity to all pairs; if coarsening distant blocks hurts, this operator is wrong here.
- **cheap_probe:** compress far-field blocks to a low-rank/multipole summary; measure reconstruction error and cost scaling vs full dense on a toy.
- **failure_signature:** reach for it when pairwise cost dominates but distant interactions are smooth/low-rank (long-context attention, N-body, potential fields).
- **transfer_targets:** long-context / speculative decoding (far tokens → low-rank summary, near tokens → full attention), 3D scene fields, systems all-to-all reduction.
- **positive_examples:** Fast Multipole Method, Boundary Element Method.
- **negative_examples / when_it_misleads:** generic "linear attention" without a hierarchical error bound is not this; if far interactions are genuinely full-rank, compression silently loses signal.
- **source_episodes:** Fast Multipole Method, Boundary Element Method (科学计算 #21,#29; report-3).

### 8. conservation-structure-ization
- **one_sentence_core_move:** bake a conserved geometric structure (symplectic form, energy/passivity, momentum) into the update rule itself, instead of minimizing local step error.
- **old_object_pattern:** a discretization/update that minimizes per-step truncation error but slowly drifts (energy, volume, momentum).
- **new_object_pattern:** a structure-preserving map (symplectic integrator / port-Hamiltonian energy-routing / variational integrator) with a modified invariant.
- **mathematical_frame:** Algebraic / group (symplectic 2-form, Dirac structure).
- **core_simplification:** long-horizon qualitative correctness stops depending on tiny step error — the conserved quantity is exact by construction, so stability over huge horizons is free.
- **differential_prediction:** predicts *bounded invariant drift over very long rollouts* and passivity-guaranteed stability of interconnected subsystems — a plain accurate integrator predicts secular drift/blow-up over long horizons even when short-step error is smaller.
- **cheap_probe:** long-horizon rollout on a conservative toy (orbit / pendulum); compare energy drift of a structure-preserving update vs a same-order plain integrator.
- **failure_signature:** reach for it when short-term accuracy is fine but long rollouts drift/explode, or when composing modules causes instability.
- **transfer_targets:** VLA world-model long rollouts, diffusion/LLM conservative update rules, multi-agent orchestration stability (energy-routed communication).
- **positive_examples:** Symplectic integrators, Variational integrators, Port-Hamiltonian systems.
- **negative_examples / when_it_misleads:** adding a soft energy-penalty loss is not structure-preservation (no exact invariant, no long-horizon-drift guarantee); a quaternion swap with a Euclidean loss doesn't complete it.
- **source_episodes:** Symplectic / Variational integrators, Port-Hamiltonian (科学计算 #17-19; report-3).

## Family 3 — Distribution geometry (canon core — DISCOUNTED, but keep the sharpest, most transferable)

### 9. ratio-ization ★
- **one_sentence_core_move:** stop fitting an absolute quantity; fit the local *ratio* "which is more like data / more preferred / more acceptable."
- **old_object_pattern:** a normalized density, a single-point score, a hard label, or an explicit scalar reward.
- **new_object_pattern:** a log-density ratio / logit margin / Bradley-Terry preference ratio / accept-reject ratio.
- **mathematical_frame:** Probability / Markov (density → likelihood ratio, Bradley-Terry).
- **core_simplification:** drops the partition function, global calibration, and absolute-reward scaling — only the *local comparison* remains.
- **differential_prediction:** predicts *rank flips, pairwise preference, and acceptance rate that stay stable even when absolute scores drift* (incl. OOD) — an absolute-value model predicts a calibrated scalar but not a stable local ordering.
- **cheap_probe:** freeze the backbone, train a tiny binary head on chosen/rejected or data/noise pairs; check AUC, rank consistency, and OOD-slice rank transfer.
- **failure_signature:** reach for it when absolute scores are hard to calibrate or drift, but relative ordering is what the downstream mechanism actually consumes.
- **transfer_targets:** speculative-decoding (accept/reject IS a local ratio), preference alignment, dataset difficulty scoring, distillation (relative teacher preference).
- **generation_test (spec-decode):** *candidate* — train a small draft-vs-target *acceptor head* on frozen hidden states predicting P(target accepts token) as a learned ratio, replacing the logit-gap rule. *Differential:* the learned acceptor raises accepted length over the fixed logit-ratio criterion at matched target-calls, especially on OOD slices where absolute logits drift. *Cheap probe:* fit the acceptor on logged draft/target pairs and compare its accept-decisions to the logit rule offline. Kill if it never beats the logit gap.
- **positive_examples:** Noise-Contrastive Estimation, DPO, CPC/InfoNCE, SimCLR.
- **negative_examples / when_it_misleads:** swapping CE for a pairwise loss with no interpretable ratio object, or hard-negative mining alone, is not ratio-ization.
- **source_episodes:** NCE (report-2/3), DPO (AI案例 C; report-2), CPC/SimCLR (report-2), Speculative Decoding accept-rate (AI案例 table).

### 10. correction-field-ization
- **one_sentence_core_move:** when predicting the clean endpoint is too hard, predict instead the local *direction that pushes a corrupted sample back toward the clean manifold*.
- **old_object_pattern:** a normalized density, a one-shot generator, or a single-point regression to the answer.
- **new_object_pattern:** a score / correction vector field: "given this corrupted state, which way is cleaner."
- **mathematical_frame:** Dynamical systems & control (score / correction vector field).
- **core_simplification:** cancels the normalization constant and turns a hard global fit into a local "which way to nudge," which is learnable and composable.
- **differential_prediction:** predicts a *local correction direction and a noise-level-dependent recoverability* — a direct endpoint regressor collapses multimodal targets to a mean and predicts no "which way to fix" signal.
- **cheap_probe:** train a small score/correction head on a toy bimodal or add-noise-then-recover task; measure angle between predicted direction and the oracle correction, and mode coverage vs a mean regressor.
- **failure_signature:** reach for it when a one-shot predictor mean-collapses on multimodal targets, or when you have a natural corruption process.
- **transfer_targets:** diffusion-LLM (reverse token correction), VLA multimodal action heads, MLLM 3D grounding as a query correction field.
- **positive_examples:** Denoising Score Matching, SMLD, diffusion score, Diffusion Policy.
- **negative_examples / when_it_misleads:** calling any critic/logit "score" is relabeling unless it forms an actual reverse-correction field; canon — low marginal novelty.
- **source_episodes:** DSM / Score Matching / SMLD (AI案例 A; report-2), Diffusion Policy (机器人 #1).

### 11. path-straightening ★
- **one_sentence_core_move:** replace an arbitrarily curved generation/control trajectory with the straightest (near-geodesic) transport path between endpoints.
- **old_object_pattern:** any curved sampling/rollout path that "gets there" but needs many small steps.
- **new_object_pattern:** a low-curvature displacement-interpolation / rectified transport path.
- **mathematical_frame:** Optimal transport (displacement geodesic).
- **core_simplification:** removes wasted curvature, so error amplification per large step drops and few-step integration becomes accurate.
- **differential_prediction:** predicts *graceful quality degradation at few steps* and a measurable curvature↔few-step-error (or ↔accept-rate) relationship — a curved-path model degrades sharply as steps drop.
- **cheap_probe:** on a small model, correlate path curvature / ODE residual with 1/2/4-step performance.
- **failure_signature:** reach for it when quality falls off a cliff as you cut sampling/rollout steps, or when different valid paths to the same target behave very differently.
- **transfer_targets:** diffusion-LLM few-step block decoding, speculative generation, VLA continuous-action rollout, scientific ML.
- **generation_test (diffusion-LLM):** *candidate* — add a straightness penalty to a block-diffusion LLM's hidden-state transport so a 2-step block decode approaches an 8-step one. *Differential:* accepted-block length at 2 steps rises toward the 8-step ceiling as curvature drops, with a monotone curvature↔accept correlation. *Cheap probe:* measure per-block path curvature vs low-step verification pass-rate before training the penalty. Kill if no correlation.
- **positive_examples:** Rectified Flow, Benamou–Brenier displacement path, OT-path Flow Matching.
- **negative_examples / when_it_misleads:** a generic smoothness regularizer with no "fewer-steps-is-better" differential is not path-straightening.
- **source_episodes:** Rectified Flow / Flow Matching (AI案例 A; report-2), Benamou-Brenier (科学计算 #11; report-3).

### 12. transport-coupling-ization ★
- **one_sentence_core_move:** rewrite a matching / similarity / divergence problem as "how mass flows from A to B under a cost geometry" — a coupling object, possibly across different spaces.
- **old_object_pattern:** JS/KL divergence, cosine similarity, Hungarian/heuristic hard matching, direct pose regression.
- **new_object_pattern:** a transport plan / soft coupling / Gromov-Wasserstein cross-space correspondence.
- **mathematical_frame:** Optimal transport (coupling, Wasserstein / Gromov-Wasserstein).
- **core_simplification:** support mismatch, shape misalignment, and unstable hard matching are handed to a single differentiable coupling object.
- **differential_prediction:** predicts *mass coupling, support-mismatch sensitivity, one-to-many alignment, and — for Gromov-Wasserstein — alignment across spaces of different dimension using only internal distance structure*. Cosine/Hungarian predict a match but no transport geometry or cross-space capability.
- **cheap_probe:** compute a Sinkhorn / approximate-GW plan on small data; measure alignment recall and support-mismatch robustness vs cosine/Hungarian.
- **failure_signature:** reach for it when similarity is high but alignment is wrong, supports mismatch, or you must align two heterogeneous modalities/spaces.
- **transfer_targets:** VLA multimodal fusion (align video↔action↔language segments), dataset dedup/risk slicing, distillation as coupling, OOD detection.
- **generation_test (VLA fusion):** *candidate* — align video clips, action chunks, and language skill descriptions with a Gromov-Wasserstein coupling (each in its own space) instead of a shared-embedding cross-attention. *Differential:* GW recovers correct cross-modal correspondences under support mismatch (unequal clip/skill counts) where cosine cross-attention fails, and needs no shared embedding dim. *Cheap probe:* run GW on frozen per-modality features, measure alignment recall vs cross-attention on held-out pairs. Kill if cosine already suffices.
- **positive_examples:** WGAN, Sinkhorn OT, Gromov-Wasserstein, Transporter Networks, Word-Mover's-Distance.
- **negative_examples / when_it_misleads:** naming a loss "Wasserstein" with no coupling/flow object is a killed transplant.
- **source_episodes:** WGAN/Sinkhorn/GW (科学计算 #14-16; report-2/3), Transporter (机器人 #17).

### 13. particle-measure-ization
- **one_sentence_core_move:** replace a single model / full covariance / analytic posterior with a small set of interacting particles (an empirical measure).
- **old_object_pattern:** one point estimate, a single Gaussian belief, a full covariance matrix, or one posterior parameterization.
- **new_object_pattern:** an ensemble / particle empirical measure whose spread and interaction carry the uncertainty and geometry.
- **mathematical_frame:** Statistical mechanics / mean-field (interacting-particle empirical measure).
- **core_simplification:** replaces an intractable full covariance / exact posterior with a handful of samples that approximate it and deform naturally.
- **differential_prediction:** predicts that *ensemble spread tracks epistemic error / OOD / planning-regret* and that particles capture topology change and deformation a single Gaussian cannot. A point estimate predicts confident single answers uncorrelated with failure.
- **cheap_probe:** 5–20 small models / particles; correlate spread with error, OOD flag, or planning regret.
- **failure_signature:** reach for it when a single model is confidently wrong, covariances are too big to operate on, or geometry deforms (free surface, large strain).
- **transfer_targets:** speculative-decoding accept-risk from a small draft ensemble, VLA belief as particles, systems uncertainty tracking, OOD.
- **positive_examples:** SVGD, Ensemble Kalman Filter/Inversion, PETS, SPH, Material Point Method.
- **negative_examples / when_it_misleads:** MC-dropout or bagging-for-accuracy is not this unless the measure itself enters the decision/inference object.
- **source_episodes:** SVGD (AI案例 A), EnKF/EKI (科学计算 #36; report-3), PETS/PILCO (report-4), SPH/MPM (report-3).

## Family 4 — Structure rewrites (element → composable structure object)

### 14. set-ization
- **one_sentence_core_move:** rewrite a flat, falsely-ordered, fixed-length input as a permutation-invariant, variable-cardinality set.
- **old_object_pattern:** a fixed-length ordered vector / concatenated list with an imposed order.
- **new_object_pattern:** an exchangeable set function (pool over element embeddings).
- **mathematical_frame:** Algebraic / group (permutation-invariant set function).
- **core_simplification:** removes the fake-order and fixed-count assumptions so reordering and count changes stop being read as signal.
- **differential_prediction:** predicts *permutation robustness and cardinality generalization* (add/remove elements) — a sequence model predicts order- and length-sensitivity.
- **cheap_probe:** Deep-Sets probe on existing features; shuffle order and extrapolate element count; watch performance/calibration.
- **failure_signature:** reach for it when performance drops on reordering, or when the number of objects/tools/items varies.
- **transfer_targets:** multimodal fusion of variable object/tool lists, VLA object sets, agent memory/tool routing.
- **positive_examples:** Deep Sets, Slot Attention (as set), FOCUS.
- **negative_examples / when_it_misleads:** mean-pooling alone with no permutation/cardinality differential is not set-ization.
- **source_episodes:** Deep Sets (report-2), Slot Attention/FOCUS (机器人; report-4).

### 15. graph-ization ★
- **one_sentence_core_move:** rewrite a flat global state as an explicit entity–relation graph with message passing.
- **old_object_pattern:** a single global embedding / concatenated Euclidean features / one monolithic state vector.
- **new_object_pattern:** nodes + edges + messages (scene graph, interaction graph, dependency graph).
- **mathematical_frame:** Algebraic / group (relational graph, message-passing operator).
- **core_simplification:** non-local dependence is pushed onto relation edges, so a global vector no longer has to memorize all local structure.
- **differential_prediction:** predicts *edge-intervention effects, object-count extrapolation, and relational-composition generalization* — a global embedding predicts none of these and breaks when object count changes.
- **cheap_probe:** attach detections/entities to a graph on a frozen backbone; do edge intervention, drop-one-node, count extrapolation.
- **failure_signature:** reach for it when a flat model fails to compose over relations or breaks as entity count changes, or you need failure localization.
- **transfer_targets:** VLA long-horizon rearrangement (scene-graph edit), MLLM grounding, systems/agent dependency graphs, multi-object world models.
- **generation_test (VLA fusion / planning):** *candidate* — represent the manipulation scene as a contact/relation scene graph and pose the goal as a graph edit, instead of a monolithic state → plan. *Differential:* graph-edit planning generalizes to +N distractor objects and localizes failures to specific edges, where a monolithic-state policy degrades with object count. *Cheap probe:* frozen detector → scene graph → measure planning stability vs object count and whether edge edits are diagnosable. Kill if a flat state matches it.
- **positive_examples:** MPNN, Interaction Networks, Sequential Manipulation Planning on Scene Graph, SayPlan / HOV-SG scene graphs.
- **negative_examples / when_it_misleads:** feeding an adjacency matrix as an extra feature is not graph-ization; a GNN with no explicit relation object is decoration.
- **source_episodes:** MPNN (report-2), Interaction Networks/OODP (report-4), scene-graph planning / SayPlan / HOV-SG (机器人; report-4).

### 16. object-slot-ization ★
- **one_sentence_core_move:** force a tangled scene representation into a set of exchangeable object slots that persist and bind independently.
- **old_object_pattern:** a single feature map / global latent / whole-image world-model state.
- **new_object_pattern:** competitive object slots / factorized per-object latents with identity persistence.
- **mathematical_frame:** Algebraic / group (exchangeable object slots).
- **core_simplification:** object binding, occlusion, and re-arrangement are isolated to per-object latents instead of one global vector absorbing everything.
- **differential_prediction:** predicts *object-level intervention, compositional OOD (novel object combinations), occlusion re-identification, and object permanence* — a global latent predicts entangled failures and identity drift under occlusion/rearrangement.
- **cheap_probe:** frozen perceptual backbone + slot probe; swap/occlude objects, extrapolate count, drop a counterfactual object.
- **failure_signature:** reach for it when a model fails on occlusion, rearrangement, or multi-object compositions, or when object identity drifts over time.
- **transfer_targets:** VLA world models & grounding, MLLM multi-object binding, OOD compositional generalization.
- **generation_test (VLA fusion):** *candidate* — replace a monolithic RSSM latent with a slot-RSSM and bind language tokens to slots. *Differential:* slot-RSSM keeps object-tracking accuracy under occlusion/rearrangement where the monolithic latent's identity drifts, and language-to-slot binding survives distractors. *Cheap probe:* on frozen video features, compare slot vs global latent on an occlusion re-ID probe before any policy training. Kill if slots don't beat global on persistence.
- **positive_examples:** Slot Attention, FOCUS, SOLD, OODP, COMPAS.
- **negative_examples / when_it_misleads:** DETR queries / region tokens that don't carry object persistence are an implementation detail, not slot-ization.
- **source_episodes:** Slot Attention (AI案例 C; 机器人), FOCUS/SOLD/COMPAS (机器人; report-4).

### 17. field-ization
- **one_sentence_core_move:** replace a discrete geometry / label mask / box with a continuously queryable field.
- **old_object_pattern:** voxels/mesh/box/class-mask, discrete pose candidates.
- **new_object_pattern:** a continuous field: radiance / language-embedding / descriptor / signed-distance, queryable at any point.
- **mathematical_frame:** Dynamical systems & control (continuous field / implicit function).
- **core_simplification:** removes resolution/topology/fixed-label boundaries — geometry and semantics live in one continuous, differentiable carrier.
- **differential_prediction:** predicts *arbitrary-point queries, spatial gradients for optimization, open-vocabulary grounding, and continuous-scale interpolation* — a discrete grid/box predicts only at fixed cells and breaks off-lattice.
- **cheap_probe:** train a tiny field head on a frozen backbone; test arbitrary-point query, sub-voxel interpolation, open-vocab / novel-view recall.
- **failure_signature:** reach for it when discrete geometry tears at boundaries, or you need open-vocabulary / continuous queries.
- **transfer_targets:** MLLM 3D grounding, VLA scene representation, scientific ML implicit solutions.
- **positive_examples:** NeRF, LERF, Neural Descriptor Fields, iMAP, ConceptFusion.
- **negative_examples / when_it_misleads:** a denser voxel grid is not field-ization unless it supports continuous query/gradient; canon-adjacent, discount.
- **source_episodes:** NeRF (AI案例; report-2), LERF/NDF/iMAP/ConceptFusion (机器人; report-4).

### 18. affordance-field-ization ★
- **one_sentence_core_move:** replace "what is this object" with a spatial field of "where and how can it be acted on (and where it can't)."
- **old_object_pattern:** object category, a fixed primitive library, or a global pose, feeding a recognize-then-plan cascade.
- **new_object_pattern:** an actionability / affordance / constraint-value field over interaction points.
- **mathematical_frame:** Dynamical systems & control (spatial value / affordance field).
- **core_simplification:** collapses the recognize→plan cascade — task-relevant complexity moves forward into a local action-site field.
- **differential_prediction:** predicts *contact/action sites and turns language constraints into spatially executable conditions*, exposing failure earlier than a category+pose pipeline (which can identify an object yet still not know where to act).
- **cheap_probe:** train an interaction-point heatmap / value map on frozen perception; test geometry perturbation, open-vocab targets, closed-loop replanning.
- **failure_signature:** reach for it when the model recognizes an object but still can't act, or a fixed primitive library can't localize where to apply it.
- **transfer_targets:** VLA manipulation, MLLM grounding (emit a field not a text plan), multimodal fusion of language + geometry into one carrier.
- **generation_test (VLA fusion):** *candidate* — have the MLLM emit a 3D affordance/constraint field directly (à la VoxPoser) instead of a text plan, fusing language + vision + geometry into one queryable value map. *Differential:* the field composes language constraints ("avoid X", "grasp handle") additively and supports closed-loop replanning under perturbation, where a text plan must re-decode. *Cheap probe:* compare category-first vs affordance-first on a small grasp set under geometric perturbation. Kill if category+pose matches it.
- **positive_examples:** Where2Act, AO-Grasp, VoxPoser, CLIPort, Contact-GraspNet.
- **negative_examples / when_it_misleads:** visualizing a saliency map as "affordance" while control still uses primitive lookup is a killed transplant.
- **source_episodes:** Where2Act/VoxPoser/CLIPort/Contact-GraspNet/AO-Grasp (机器人; report-4).

### 19. correspondence-field-ization ★
- **one_sentence_core_move:** recast action/pose regression as solving a spatial *correspondence* (from-where to-where) via feature matching / detection on a grid.
- **old_object_pattern:** direct SE(3)/action-vector regression from a global feature.
- **new_object_pattern:** a pick-place correspondence / spatial transport field / voxel action detection.
- **mathematical_frame:** Optimal transport (spatial correspondence).
- **core_simplification:** turns "where to move what" from global-coordinate regression into local matching + displacement, inheriting spatial equivariance.
- **differential_prediction:** predicts *one-to-many feasible sites, spatial-equivariant transfer, and robustness on symmetric/deformable objects* — a single-point regressor collapses on symmetry, occlusion, and cloth/rope.
- **cheap_probe:** on a toy relocation task, train only a frozen-feature correlation/detection head; test equivariance, occlusion robustness, multi-target recall.
- **failure_signature:** reach for it when action regression fails on symmetry, deformables, or needs spatial generalization to new layouts.
- **transfer_targets:** VLA spatial action heads, MLLM grounding, UI agents (screen-region correspondence vs click token).
- **generation_test (VLA fusion):** *candidate* — give an embodied model a correspondence/detection head over the 3D scene (à la Transporter/PerAct) instead of an action MLP. *Differential:* correspondence transfers to novel object poses/layouts with equivariance and represents multimodal placements, where the MLP regressor picks one mode and fails on symmetry. *Cheap probe:* frozen feature correlation head on toy relocation; measure equivariant transfer vs an MLP head. Kill if the head degenerates to a regressor with no gain.
- **positive_examples:** Transporter Networks, PerAct, Neural Descriptor Fields, Contact-GraspNet.
- **negative_examples / when_it_misleads:** visualizing attention while execution still uses a downstream regressor is not correspondence-field-ization.
- **source_episodes:** Transporter (机器人 #17; report-4), PerAct/NDF (report-4).

### 20. operator-lifting ★
- **one_sentence_core_move:** lift a single instance's input→output map to a *function-to-function operator* over an entire family, so one trained object serves the whole family.
- **old_object_pattern:** point-to-point regression on a fixed grid; solve/fit each instance from scratch.
- **new_object_pattern:** a nonlinear operator (branch/trunk, spectral kernel) mapping input functions to output functions.
- **mathematical_frame:** Spectral / operator (function-to-function operator).
- **core_simplification:** the "re-solve every time the boundary/parameter changes" cost moves into one shared, reusable solution operator.
- **differential_prediction:** predicts *cross-resolution, cross-boundary-condition, cross-instance generalization from one forward pass* — a single-instance surrogate predicts only its training grid and must retrain per instance/resolution.
- **cheap_probe:** on a small PDE/control toy, hold out unseen boundary-conditions/resolutions; compare a state-predictor vs an operator-predictor.
- **failure_signature:** reach for it when each new condition/resolution forces a retrain, or you want zero-shot super-resolution / family-level reuse.
- **transfer_targets:** diffusion-LLM (a decoding operator over prompt-conditioned families / resolutions), MLLM cross-modal maps, VLA policy operators, distillation as operator matching.
- **generation_test (diffusion-LLM):** *candidate* — learn "prompt-context function → per-position denoising operator" so one operator generalizes across sequence lengths/resolutions instead of a fixed-length head. *Differential:* zero-shot transfer to unseen sequence lengths (resolution-invariance) that a fixed-length decoder lacks. *Cheap probe:* train a branch/trunk head on short sequences, test length extrapolation vs a fixed head. Kill if it doesn't extrapolate.
- **positive_examples:** DeepONet, Fourier Neural Operator, Operator Inference, DIMON, PINO.
- **negative_examples / when_it_misleads:** multi-in/multi-out regression with no family-level generalization is a bigger net, not an operator.
- **source_episodes:** DeepONet/FNO (AI案例 D; 科学计算; report-2/3), Operator Inference/DIMON/PINO (report-3).

## Family 5 — Sparsity & coding (long-tail: change the object so the algorithm becomes feasible)

### 21. sparse-support-ization
- **one_sentence_core_move:** model a dense signal/function as a *sparse support* (which few coefficients/terms are nonzero) in a chosen basis or library.
- **old_object_pattern:** a dense signal fit under Nyquist-style sampling, or a black-box function approximator.
- **new_object_pattern:** sparse coefficients / an active-term support set (compressed-sensing recovery, SINDy library).
- **mathematical_frame:** Rate-distortion / coding (sparse support, restricted isometry).
- **core_simplification:** measurement/parameter cost drops from "signal dimension" to "effective sparsity," and the answer becomes *which* terms exist.
- **differential_prediction:** predicts *recoverability from few measurements ∝ sparsity, not dimension*, and identifies the specific active support — a dense fit predicts you need ~dimension measurements and yields no interpretable support.
- **cheap_probe:** L1 recovery from sub-sampled measurements vs a dense fit on a toy; check exact-support recovery and measurement count.
- **failure_signature:** reach for it when the true structure is few-active-terms, measurements are expensive, or you want a discovered mechanism not a black box.
- **transfer_targets:** context/KV compression, dataset pruning to an active subset, mechanism discovery in dataset-eval, sparse attention.
- **positive_examples:** Sparse Coding, Compressed Sensing, SINDy.
- **negative_examples / when_it_misleads:** an L1 penalty that never yields a stable support, or "sparsity" with no few-measurement-recovery differential, is decoration.
- **source_episodes:** Sparse Coding/ICA (report-2), Compressed Sensing (科学计算 #37; report-3), SINDy (report-3).

### 22. polarization-ization ★
- **one_sentence_core_move:** apply a recursive transform that splits a medium-reliability ensemble into *extremal* sub-channels — near-perfect vs near-useless — then route the payload onto the reliable ones and freeze the rest.
- **old_object_pattern:** a uniform medium-reliability channel/ensemble decoded/handled all at once.
- **new_object_pattern:** polarized sub-channels: a deterministic-reliable set + a frozen near-random set.
- **mathematical_frame:** Rate-distortion / coding (channel polarization).
- **core_simplification:** replaces a globally chaotic medium-noise decode with a clean split — you only work hard on the reliable fraction and pre-set the rest.
- **differential_prediction:** predicts that after the transform, per-unit reliability *bimodalizes* (a sharp two-peak split), so a fixed frozen set becomes safe — an untransformed ensemble predicts a unimodal medium-reliability spread with no freezable subset.
- **cheap_probe:** apply a polarizing (recursive Kronecker) transform to token/logit positions; measure whether per-position confidence bimodalizes and whether freezing the low-reliability set is lossless.
- **failure_signature:** reach for it when a medium-confidence ensemble has no obvious high-certainty subset to exploit but you suspect one can be *induced*.
- **transfer_targets:** speculative-decoding (polarize positions into "commit cheaply" vs "must verify"), knowledge distillation (route deterministic vs high-entropy tokens), sparse computation.
- **generation_test (spec-decode):** *candidate* — apply a polar-style recursive transform over a token block so positions split into high-certainty (draft commits, no verify) and low-certainty (always verify) sets. *Differential:* per-position confidence bimodalizes after the transform, letting the high-certainty set skip verification with bounded error — a raw block shows no such freezable set. *Cheap probe:* offline, apply the transform to logged logits and test whether a frozen high-certainty set is near-lossless. Kill if confidence stays unimodal.
- **positive_examples:** Polar codes (channel polarization).
- **negative_examples / when_it_misleads:** simply thresholding confidence is not polarization (no transform that *induces* the split); calling a gate "polar" is relabeling.
- **source_episodes:** Polar Codes (科学计算 #33; report-3 explicitly flags spec-decode transfer).

## Family 6 — Invariance & symmetry (move the nuisance onto an explicit axis)

### 23. invariant-ization ★
- **one_sentence_core_move:** replace "average fit over a pooled mixture" with a predictor/representation constrained to be *stable across environments* (or a causal parent set).
- **old_object_pattern:** pooled ERM that silently absorbs environment-specific shortcuts.
- **new_object_pattern:** an invariant predictor / causal parent set / environment-invariant latent, with environment as an explicit index.
- **mathematical_frame:** Causal graphs (cross-environment invariant predictor).
- **core_simplification:** shortcut complexity is moved onto an explicit "environment" dimension instead of being averaged into the weights.
- **differential_prediction:** predicts *cross-environment failure and worst-domain performance* — pooled ERM reports only an average and hides which features are environment-specific; a counterfactual feature swap changes predictions for spurious features only.
- **cheap_probe:** environment split + linear/IRM probe; check worst-domain lift and counterfactual feature-swap sensitivity.
- **failure_signature:** reach for it when pooled training looks good but collapses on a new environment, and errors concentrate on environment-specific shortcuts.
- **transfer_targets:** VLA/multimodal fusion (environment-invariant fusion bottleneck), dataset curation, clinical/OOD robustness.
- **generation_test (VLA fusion):** *candidate* — learn a multimodal fusion bottleneck penalized to be invariant across embodiment/sensor "environments" instead of pooling all data. *Differential:* worst-embodiment performance rises and env-specific sensor artifacts get suppressed (counterfactual swap invariant), where pooled fusion overfits to the dominant embodiment. *Cheap probe:* split by embodiment, IRM probe on frozen features, measure worst-group lift. Kill if invariant features only work in-distribution.
- **positive_examples:** IRM, Invariant Causal Prediction, Group DRO, Clinical Foundation Models.
- **negative_examples / when_it_misleads:** plain augmentation-robustness with no environment index is easily disguised regularization, not invariant-ization.
- **source_episodes:** IRM/ICP (report-2/3), Clinical Foundation Models (AI案例 E), Group DRO (report-2).

### 24. group-equivariance-ization
- **one_sentence_core_move:** replace topology-broken Euclidean coordinates/channels with objects that respect the problem's symmetry group.
- **old_object_pattern:** Euler angles / Euclidean pose / independently-processed scalar channels (equivariant only to translation).
- **new_object_pattern:** a Lie-group / group-equivariant feature field / geometric (multivector) object.
- **mathematical_frame:** Algebraic / group (group representation, equivariant map).
- **core_simplification:** rotation/reflection/gauge symmetry is hard-coded into the representation, so it isn't re-learned from data and can't be violated.
- **differential_prediction:** predicts *rotation continuity (no gimbal-lock discontinuity), bounded equivariance error, and coordinate-change-stable outputs* — a Euclidean model predicts wrap-around discontinuities and needs symmetry taught via augmentation.
- **cheap_probe:** synthetic rotation/gauge perturbation set; compare Euclidean vs group/manifold representation on continuity and OOD-rotation error.
- **failure_signature:** reach for it when models break at rotation boundaries, pose wrap-around, or coordinate-frame changes.
- **transfer_targets:** VLA 6-DoF action heads (SE(3) diffusion), MLLM 3D grounding, scientific ML (equivariant PDE fields).
- **positive_examples:** Grasp Diffusion Network (SE(3)), Group-Equivariant CNN, Clifford Neural Layers, PP-GraspNet.
- **negative_examples / when_it_misleads:** dropping quaternions into a net while keeping a Euclidean loss does not complete the object shift.
- **source_episodes:** GDN (机器人 #3), Group-Equivariant CNN/Clifford (report-2/3), PP-GraspNet (机器人).

## Family 7 — Sufficiency, uncertainty & representation-output objects

### 25. belief-ization ★
- **one_sentence_core_move:** compress observation history into a planning-*sufficient* belief/latent carrying uncertainty, and plan on that object rather than on raw observations.
- **old_object_pattern:** reactive observation policy, stacked history, or an explicit but uncertainty-free state.
- **new_object_pattern:** a posterior belief / value-equivalent latent / RSSM state as a sufficient statistic.
- **mathematical_frame:** Probability / Markov (Bayesian belief / sufficient statistic).
- **core_simplification:** long-history dependence and hidden-state uncertainty collapse into a compact sufficient statistic instead of being re-derived each step.
- **differential_prediction:** predicts *the value of information-gathering actions and how hidden-parameter uncertainty affects control* — a reactive/history-stack policy predicts no info-value and fails on POMDP structure.
- **cheap_probe:** oracle-state substitution + belief probe: if a belief head approaches oracle-state performance while a history stack does not, it's real.
- **failure_signature:** reach for it when failures stem from hidden state / partial observability rather than backbone capacity, and info-gathering matters.
- **transfer_targets:** speculative-decoding (a draft prefix as an uncertainty-bearing belief, not fixed text), VLA occlusion, systems tool-state belief, OOD.
- **generation_test (spec-decode):** *candidate* — treat the draft prefix as an uncertainty-bearing belief over continuations rather than committed text; the target updates the belief and only "observes" (verifies) high-value positions. *Differential:* verifying by information-value accepts more tokens per target-call than left-to-right verification, because low-info positions are committed under the belief. *Cheap probe:* on logged runs, test whether per-position entropy predicts which verifications actually flip a token. Kill if entropy is uninformative about flips.
- **positive_examples:** Kalman Filter, POMDP belief state, PSR, PlaNet/Dreamer RSSM, MuZero value-equivalent latent, RMA extrinsics.
- **negative_examples / when_it_misleads:** any RNN hidden state is not a belief unless it carries uncertainty and enters planning.
- **source_episodes:** Kalman/POMDP/PSR (report-3/4), RMA (机器人 #26), PlaNet/Dreamer/MuZero (report-4).

### 26. distribution-output-ization
- **one_sentence_core_move:** make the output object a distribution/set over answers instead of a single point or expectation.
- **old_object_pattern:** a single action, an expected return, or one demonstration trajectory.
- **new_object_pattern:** an action/trajectory/return distribution (or a stochastic policy law).
- **mathematical_frame:** Probability / Markov (output distribution).
- **core_simplification:** future uncertainty is no longer squashed to a mean; multimodality and risk become first-class, controllable objects.
- **differential_prediction:** predicts *risk profile, multimodal coverage, and via-point conditioning* — a point/expectation predictor hides risk and mean-collapses on multimodal targets.
- **cheap_probe:** distribution head on a toy multimodal control task; measure mode coverage and a risk-sensitive metric vs a point head.
- **failure_signature:** reach for it when a point predictor mean-collapses, or tail risk is hidden by an expectation.
- **transfer_targets:** VLA action chunks as trajectory distributions, distillation, risk-aware tool use, OOD.
- **positive_examples:** Distributional RL, ProMP, Diffusion Policy, SAC (max-entropy policy).
- **negative_examples / when_it_misleads:** adding a Gaussian variance head with no risk/multimodal-control use is not this.
- **source_episodes:** Distributional RL (report-2), ProMP/Diffusion Policy (机器人; report-4), SAC (AI案例; report-2).

### 27. coverage-set-ization
- **one_sentence_core_move:** make the output a *set with a guaranteed coverage rate* instead of a point prediction or an uncalibrated probability.
- **old_object_pattern:** a single prediction or a raw softmax "confidence."
- **new_object_pattern:** a set-valued predictor calibrated (via nonconformity scores) to contain the truth at a chosen rate.
- **mathematical_frame:** Probability / Markov (conformal / distribution-free coverage).
- **core_simplification:** turns "how uncertain" from a post-hoc diagnostic into the primary, directly-controllable output object.
- **differential_prediction:** predicts a *finite-sample, distribution-free coverage rate you can dial* (set size grows on hard inputs) — a point/softmax model gives no coverage guarantee and its confidence is uncalibrated under shift.
- **cheap_probe:** split-conformal wrapper on any frozen model; check empirical coverage vs target and set-size vs difficulty. (Trivially cheap — no training.)
- **failure_signature:** reach for it when you must bound error rate / abstain reliably rather than optimize an average score.
- **transfer_targets:** LLM abstention, dataset-eval / auditing, OOD-safe deployment, spec-decode accept thresholds with coverage control.
- **positive_examples:** Conformal Prediction.
- **negative_examples / when_it_misleads:** a fixed confidence threshold is not conformal (no distribution-free coverage guarantee).
- **source_episodes:** Conformal Prediction (report-2/3).

### 28. latent-variable-ization
- **one_sentence_core_move:** compress a complex observation object into an inferable, interpolatable, conditionable latent variable with a posterior.
- **old_object_pattern:** observation-space regression, per-task independent fitting, or a fully explicit state.
- **new_object_pattern:** a latent-variable generative process + amortized posterior (ELBO), or a context-conditioned function law.
- **mathematical_frame:** Probability / Markov (latent-variable posterior, ELBO).
- **core_simplification:** high-dim observation complexity and per-instance posterior inference move into an encoder/posterior family, cutting instance-level solve cost.
- **differential_prediction:** predicts *interpolation, missing-data imputation, counterfactual sampling, and few-shot task adaptation* — direct observation-space regression predicts none of these and can't sample/condition.
- **cheap_probe:** small latent bottleneck + posterior-collapse diagnostic; test latent traversal semantics, imputation, few-shot adaptation vs a deterministic baseline.
- **failure_signature:** reach for it when you need conditional generation / imputation / few-shot but the observation-space model is unstable.
- **transfer_targets:** VLA latent plans, scientific ML surrogates, distillation, OOD.
- **positive_examples:** VAE, Variational Inference, Neural Processes, Latent ODE, PEARL.
- **negative_examples / when_it_misleads:** calling any hidden state "latent" with no inference/imputation/sampling capability is relabeling; canon — discount.
- **source_episodes:** VAE/VI (report-2), Neural Processes (AI案例 B), Latent ODE/PEARL (report-3/4).

### 29. information-bottleneck-ization
- **one_sentence_core_move:** replace "keep all input information" with "keep the minimal representation sufficient for the target," trading compression against predictiveness.
- **old_object_pattern:** a representation that tries to reconstruct/preserve the full input.
- **new_object_pattern:** a compressed code at a chosen point on the rate–relevance tradeoff.
- **mathematical_frame:** Rate-distortion / coding (information bottleneck).
- **core_simplification:** explicitly discards target-irrelevant input detail instead of carrying it, giving a measurable sufficiency/minimality knob.
- **differential_prediction:** predicts *which information to drop and OOD robustness gained by dropping environment-correlated dims* — a reconstruct-everything objective predicts no principled drop and stays sensitive to nuisance detail.
- **cheap_probe:** vary bottleneck capacity; measure task-relevant vs nuisance mutual information and downstream robustness.
- **failure_signature:** reach for it when nuisance/shortcut features hurt robustness, or you need a measurable sufficiency criterion.
- **transfer_targets:** multimodal fusion feature selection, dataset summarization, distillation, systems telemetry compression.
- **positive_examples:** Information Bottleneck, Deep Variational IB, CPC.
- **negative_examples / when_it_misleads:** a bottleneck layer with no rate–relevance tradeoff is just a narrow layer; canon — discount.
- **source_episodes:** IB/VIB (report-2), CPC (report-2), IB (科学计算 #35).

## Family 8 — Adaptation & composition (highest near-term generativity: give the un-modeled complexity a home)

### 32. residual-ization ★
- **one_sentence_core_move:** don't model from scratch — freeze a base model/controller/solver and learn only its *systematic deviation*.
- **old_object_pattern:** a full end-to-end map, control-from-zero, or physics-from-zero.
- **new_object_pattern:** a residual dynamics / residual policy / value or simulator residual on top of a base.
- **mathematical_frame:** Dynamical systems & control (base flow + residual correction).
- **core_simplification:** the base carries the first-order structure; the hardest complexity is localized into a small delta object.
- **differential_prediction:** predicts *where the base errs and that the residual is sparse/localized* (contact, friction, distribution-shift regions) — a from-scratch model predicts diffuse error and needs far more data.
- **cheap_probe:** feed base/oracle output as input, learn a tiny residual; test error-localization and hard-region gain vs relearning the full map.
- **failure_signature:** reach for it when a base model is broadly right but severely wrong in a few regions, and from-scratch data is expensive.
- **transfer_targets:** VLA (freeze a large model, learn a contact residual — the DSpark-style hot-swap warm-start), speculative-execution correction, systems latency correctors, scientific ML.
- **generation_test (spec-decode / VLA):** *candidate* — freeze the large draft/backbone and train only a small residual head that corrects its systematic bias (e.g. the beyond-Markov residual for DSpark). *Differential:* the residual concentrates on a sparse, identifiable slice (high-lag or contact-rich tokens) and improving only it recovers most of the full-retrain gain at a fraction of params. *Cheap probe:* fit a tiny residual on frozen base outputs, check whether its magnitude is sparse/localized. Kill if the residual is dense (≈ relearning everything).
- **positive_examples:** ResNet, Residual Physics-ML, Residual RL, RWM, Reflexive World Models.
- **negative_examples / when_it_misleads:** any skip connection is not residual-ization; without a "base + delta" error semantics it's a killed transplant.
- **source_episodes:** ResNet/Residual Physics (report-2/3), Residual RL/RWM (机器人; report-4).

### 33. context-ization
- **one_sentence_core_move:** rewrite a sim-to-real gap / task shift as an online-estimable latent *context* (extrinsics), not as noise or a fixed policy.
- **old_object_pattern:** a single policy, a fixed simulator, or treating the shift as unmodeled noise.
- **new_object_pattern:** a latent context / extrinsics belief / parameter distribution estimated from a short interaction.
- **mathematical_frame:** Probability / Markov (online-estimated latent context).
- **core_simplification:** "reality differs" becomes a small inferable variable instead of an unexplained failure mode.
- **differential_prediction:** predicts *fast few-shot adaptation, clustered shift structure, and embodiment dependence* — a static-robust policy predicts uniform behavior and no rapid adaptation from a few seconds of interaction.
- **cheap_probe:** infer context from a short history; test few-shot adaptation and OOD clustering vs a static robust baseline.
- **failure_signature:** reach for it when failures come from platform/environment differences rather than backbone capacity.
- **transfer_targets:** VLA embodiment-context token (estimated from 2–5 s), systems machine-state context, OOD, personalization.
- **positive_examples:** PEARL, RMA, UP-OSI, Domain Randomization, SimOpt, VariBAD.
- **negative_examples / when_it_misleads:** a fixed one-hot task ID is not context-ization (not online-estimable); disguised as a label it doesn't adapt.
- **source_episodes:** PEARL/RMA/UP-OSI/DR/SimOpt/VariBAD (机器人; report-4).

### 34. skill-option-ization
- **one_sentence_core_move:** replace a flat primitive-action space with reusable, searchable, composable temporally-extended skills.
- **old_object_pattern:** a flat action space where every step is decided from scratch.
- **new_object_pattern:** options / skills / subtask-graph nodes / dynamical primitives as explicit modeling objects.
- **mathematical_frame:** Dynamical systems & control (temporally-extended options / attractor primitives).
- **core_simplification:** long-horizon credit assignment and search move up to the skill level, shrinking effective decision depth.
- **differential_prediction:** predicts *reusable substructure, skill-call boundaries, and cross-task transfer* — a flat policy predicts no reusable sub-structure and needs deep rollouts.
- **cheap_probe:** option-boundary discovery on offline demos/rollouts; compare skill-search vs primitive-tree sample-efficiency.
- **failure_signature:** reach for it when long-horizon tasks need too-deep rollouts and sub-behaviors clearly recur.
- **transfer_targets:** VLA skill libraries, agent-runtime skill routing, speculative decoding (reusable decode "skills"/templates).
- **positive_examples:** Options, MAXQ, DMP/LMP, Learning Latent Plans from Play, SDP, Voyager.
- **negative_examples / when_it_misleads:** a hand-assembled macro-action is not skill-ization unless the skill is an explicit learned/searchable object.
- **source_episodes:** Options/MAXQ (report-4), DMP (机器人 #19), Play-plans/SDP/Voyager (机器人; report-4).

### 35. chunk-ization ★
- **one_sentence_core_move:** replace per-step decisions with multi-step blocks / phase-conditioned primitives decided and committed together.
- **old_object_pattern:** single-step Markov action / point-by-point trajectory following / token-by-token decode.
- **new_object_pattern:** a multi-step action chunk / phase variable / block with internal consistency.
- **mathematical_frame:** Dynamical systems & control (temporal chunk).
- **core_simplification:** error accumulation and high-frequency jitter are removed from every step and replaced by block-level consistency.
- **differential_prediction:** predicts a *chunk-length ↔ compounding-error relation and temporal-stability gain* — a single-step policy predicts jitter and needs high-frequency feedback to stay stable.
- **cheap_probe:** ACT/DMP-style mini-chunk probe; compare 1-step vs k-step smoothness and compounding error.
- **failure_signature:** reach for it when long rollouts accumulate error, actions jitter, or per-step verification is the bottleneck.
- **transfer_targets:** speculative-decoding (block verification instead of token-by-token), VLA latent action chunks, diffusion-LLM block assembly.
- **generation_test (spec-decode / diffusion-LLM):** *candidate* — decode and verify a *block* with an internal-consistency prior (a learned chunk object) rather than token-by-token accept/reject. *Differential:* enforcing intra-block consistency raises accepted-block length over independent per-token acceptance at equal target-calls, and error compounds sub-linearly in block length. *Cheap probe:* measure whether within-block token correlations are high enough that a consistency prior would change accept decisions on logged runs. Kill if tokens are near-independent within a block.
- **positive_examples:** ACT, VQ-BeT, Diff-Control, ProMP.
- **negative_examples / when_it_misleads:** frame-skip / action-repeat is not chunk-ization (no block-level object with internal consistency).
- **source_episodes:** ACT (机器人 #7), VQ-BeT/Diff-Control (机器人; report-4).

### 36. sufficient-summary-functionalization
- **one_sentence_core_move:** prove that a drastically lower-dimensional summary is *sufficient*, and rewrite the intractable many-body object as a functional of that summary.
- **old_object_pattern:** an entangled many-body / high-dim coupled object (joint wavefunction, coupled micro-DOFs, oscillating micro-coefficients).
- **new_object_pattern:** a functional of a low-dim sufficient summary (electron density, potential-energy surface, homogenized coefficient).
- **mathematical_frame:** Convex duality (variational functional of a sufficient summary).
- **core_simplification:** an exponential/high-dim object collapses to a polynomial problem in the low-dim summary — because the target property provably depends only on the summary.
- **differential_prediction:** predicts that the target property is a *function of the candidate summary alone* (a collapse test passes: fixing the summary fixes the property) — a full-coupling model predicts dependence on details the summary omits.
- **cheap_probe:** collapse test — regress the target property on the candidate summary; if residual variance is near zero, the summary is sufficient. (Runs before any heavy model.)
- **failure_signature:** reach for it when a coupling is intractable but you suspect a low-dim aggregate (a density, an average, a slow field) captures what matters.
- **transfer_targets:** compressing many-agent / long-context state to a sufficient summary, dataset-eval (is property X a function of summary S?), distillation targets.
- **positive_examples:** Density Functional Theory, Born–Oppenheimer approximation, Homogenization theory.
- **negative_examples / when_it_misleads:** picking a low-dim summary with no sufficiency test is lossy compression, not this; if the collapse test fails the operator does not apply.
- **source_episodes:** DFT / Born-Oppenheimer / Homogenization (科学计算 #26-28; report-3).

### 37. implicit-topology-ization
- **one_sentence_core_move:** replace an explicit boundary/mesh with an implicit zero-level-set / order-parameter field so topology changes are automatic.
- **old_object_pattern:** front tracking, sharp-interface mesh, explicit boundary that tears on merge/split.
- **new_object_pattern:** a signed-distance / level-set / phase field whose zero-set IS the boundary.
- **mathematical_frame:** Dynamical systems & control (level-set Hamilton–Jacobi).
- **core_simplification:** merge/split/reconnection stop needing re-meshing and collision detection — the implicit field handles topology on a fixed grid.
- **differential_prediction:** predicts *curvature, topology change (merge/split), and differentiable boundary optimization* — an explicit boundary predicts a tear/failure at merge/split and gives no clean gradient through topology change.
- **cheap_probe:** toy interface evolution; compare explicit-boundary vs SDF/phase-field on a topology-change event and gradient usefulness.
- **failure_signature:** reach for it when explicit geometry breaks on contact/merge/split, or you need to optimize through topology change.
- **transfer_targets:** VLA contact boundaries as signed-distance barriers, MLLM 3D grounding as language-conditioned SDF, 3D generation topology stability.
- **positive_examples:** Level Set Method, Phase Field / Cahn–Hilliard, GIGA/TSDF.
- **negative_examples / when_it_misleads:** an occupancy/voxel map without zero-level-set semantics is not implicit-topology-ization.
- **source_episodes:** Level Set / Phase Field (科学计算 #20; report-3), GIGA (机器人).

### 38. executable-spec-ization
- **one_sentence_core_move:** rewrite a hand-tuned scalar/heuristic (reward, plan, constraint) as an LLM-generated *executable program* that is searched, evaluated, and evolved.
- **old_object_pattern:** a manually-weighted scalar reward / fixed primitive plan / hard-coded constraint.
- **new_object_pattern:** an executable reward/constraint/plan program (a code manifold) with reflection-based improvement.
- **mathematical_frame:** Algebraic / group (executable-program object).
- **core_simplification:** removes manual shaping — the objective/constraint becomes symbolically editable, recombinable, and improvable by search.
- **differential_prediction:** predicts that the objective can be *edited/recombined symbolically and improved via reflection*, and that a program can localize sub-task error a scalar reward cannot (two behaviors with equal scalar reward get different program-level diagnostics).
- **cheap_probe:** generate the reward/constraint as code; test whether reflection-based edits improve a proxy metric and whether the program separates reward-tied behaviors.
- **failure_signature:** reach for it when a scalar objective is brittle/opaque and the true spec is compositional/symbolic.
- **transfer_targets:** RL reward design, LLM-grounded planning (VoxPoser code), neuro-symbolic planners (PDDLStream), tool/agent orchestration.
- **positive_examples:** Eureka, ARCHIE, VoxPoser (code-composed value maps), PDDLStream, Logic-Geometric Programming.
- **negative_examples / when_it_misleads:** prompting an LLM for a plan string with no executable/evaluable/evolvable object is not this.
- **source_episodes:** Eureka/ARCHIE/PRM4RL (机器人; report-4), PDDLStream/LGP (report-4).
